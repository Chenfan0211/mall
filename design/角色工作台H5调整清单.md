# 角色工作台 H5 调整清单

> 适用范围：角色工作台小程序 H5，包括自提点角色和供应商角色。
> 当前阶段：产品建议拍板后记录到本清单，后续再根据清单做 UI 设计。
> 约束：本清单不直接修改 UI 原型，不替代功能清单。

### 调整项

| 编号 | 调整项 | 优先级 | UI 设计落点建议 | 产品说明 |
| --- | --- | --- | --- | --- |
| R-H5-001 | 自提点端“订单”不要叫“送货单”，统一改成“提货订单 / 用户订单”。 | P0 | 自提点底部导航、列表标题、详情标题、搜索占位文案。 | 供应商看送货单，自提点看用户提货订单。名称混用会导致培训、权限和后续数据口径混乱。 |
| R-H5-002 | 供应商端不能出现“审核通过 / 审核拒绝”采购单按钮。 | P0 | 供应商采购单列表和详情操作区。 | 供应商只能看审核状态、撤回、重新提交、查看拒绝原因，不能审核自己的采购单。采购审核归运营/采购权限。 |

| R-H5-006 | 首页增加“当前角色状态卡”。 | P1 | 自提点工作台首页、供应商工作台首页第一屏。 | 自提点展示营业中、休假中、停用风险、待处理异常；供应商展示资质、合同、押金、准入限制。先告诉角色今天能不能正常作业。 |
| R-H5-007 | 提现页增加可提现校验口径。 | P1 | 自提点佣金提现页、供应商货款提现页。 | 展示账户余额、押金占用、待审核提现、可提现金额、最低提现金额、手续费，避免角色误以为全部余额都能提现。 |

| 编号 | 不做项 | 原因 |
| --- | --- | --- |
| R-H5-N001 | 不把角色工作台做成小型运营后台。 | 自提点和供应商只处理自身日常协同，不承担平台运营审批和风控职责。 |

### UI 设计落地提醒

| 类型 | 要求 |
| --- | --- |
| 角色边界 | 同一按钮在自提点和供应商视角中不能复用成不同业务含义。 |
| 异常闭环 | 自提点/供应商提交异常后，要明确“已提交给运营处理”，不要表现成已经完成退款、补货或入库。 |
| 资金保护 | 提现页必须把“余额”和“可提现金额”分开展示。 |
| 状态口径 | 供应商状态以“已提交 / 等待仓库确认 / 仓库已入库 / 待结算”为主，不使用供应商自己确认的入库口径。 |

| R-H5-028 | 列表筛选条件要有“清空筛选”。 | P1 | 订单、履约、退货、采购单、消息、操作记录筛选条。 | 角色端不是专业后台，筛选后看不到数据时要一键回到默认今日或当前角色视图。 |

## v1.14 UI 设计交付顺序与评审口径

| 日期 | 状态 | 来源 | 说明 |
| --- | --- | --- | --- |
| 2026-06-23 | 已拍板 | 定版后收口 | 本版不新增业务能力，只用于控制后续 UI 设计交付顺序和评审口径，避免同时画太多、评审抓不住主线。 |

### UI 设计交付顺序

| 编号 | 批次 | 交付范围 | 交付要求 |
| --- | --- | --- | --- |
| R-H5-O001 | 第一优先级 | 登录页、团长首页、自提点首页、供应商首页。 | 先确认入口、身份分流、首页信息架构，再继续画深层页面。 |
| R-H5-O002 | 第二优先级 | 提货订单、门店作业、采购/到仓三条作业主链路。 | 先跑通每日作业闭环，确保列表、详情、提交、状态一致。 |
| R-H5-O003 | 第三优先级 | 佣金/货款、提现/结算、操作记录、消息中心。 | 再补资金和追踪能力，确保记录可查、口径不混。 |
| R-H5-O004 | 第四优先级 | 通用状态页、无权限页、加载失败页、主体停用页。 | 最后补齐异常和边界状态，避免 UI 定版后漏状态。 |
| R-H5-O005 | 批次评审 | 每一批 UI 交付必须先过当前批次评审，再进入下一批。 | 防止基础入口或角色边界未定，就继续堆深层页面。 |

### 每批 UI 评审必须看 5 件事

| 编号 | 评审项 | 评审口径 |
| --- | --- | --- |
| R-H5-R001 | 身份归属 | 当前页面必须明确属于团长、自提点或供应商。 |
| R-H5-R002 | 数据隔离 | 当前页面只能展示当前身份、当前主体的数据。 |
| R-H5-R003 | 状态说明 | 当前页面必须有当前状态、是否需要我处理、下一步说明。 |
| R-H5-R004 | 基础状态 | 当前页面必须覆盖加载中、无数据、失败、不可操作原因。 |
| R-H5-R005 | 范围校验 | 当前页面不能画出第一版明确不做的内容。 |
