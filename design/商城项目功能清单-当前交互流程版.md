# 商城项目功能清单（当前交互流程版）

编制日期：2026-06-15

本文档根据当前《商城项目交互原型图.html》的页面、菜单、按钮、移动端角色切换和 WMS 单据流转整理，作为后续需求确认、接口拆分、数据库设计和开发排期的功能范围依据。

## 1. 系统范围

本项目是社区团购/商城 + 进销存配一体化系统，覆盖商品建档、商品发布、用户下单、支付、积分、采购、入库、仓储作业、配送、自提点交接、售后退货、分账佣金、提现、司机费用、经营报表和系统自动任务。

| 端口 | 使用角色 | 核心职责 |
| --- | --- | --- |
| 运营后台 | 平台运营、城市运营、采购、财务、管理员 | 商品、类目、基础资料、发布、权限、订单、售后、分账、佣金、营销、积分、报表 |
| 用户小程序 / H5 | 普通用户 | 浏览商品、切换城市/自提点、领券、加购、下单、支付、查看订单、查看积分、申请售后、评价、分享 |
| 角色工作台小程序 | 自提点、供应商 | 自提点订单/收货/退货/佣金/休假；供应商送货单/商品销售/缺货/发票/提现 |
| 仓配移动端 H5 | 收货员、采购员、拣货员、复检装车员、司机、仓管 | 采购、收货、质检、上架、拣货、缺货、复检、装车、配送、退货、库存、盘点 |
| WMS 后台 | 仓库主管、仓配运营、物流调度 | 单据流转、采购、入库、出库、配送、车辆、线路、退货、司机费用、库存、日志 |
| 系统任务 | 定时任务、MQ、缓存、搜索服务 | 商品 ES 同步、Redis 缓存刷新、自动取消订单、生成配送数据、分账重试、消息通知 |

## 2. 全链路业务流程

1. 基础资料维护：后台创建供应商、自提点、采购员、城市、仓库、库区、库位，并生成对应登录账号和数据权限。
2. 商品建档：运营维护商品、品牌、供应商、采购员、展示类目、统计类目、商品规格、库存规格、换算系数、城市价格、长宽高、重量、榜单标签、联想词。
3. 商品发布：运营创建发布模板，设置销售时间、配送日、城市/自提点白名单、城市价格、采购仓，确认发布后刷新小程序商品列表。
4. 用户下单：用户在小程序浏览首页、分类、活动、榜单、商品详情，加入购物车，提交订单，选择优惠券和收货人，完成支付。
5. 订单履约：支付成功后生成订单、支付流水、佣金预估和下单返积分记录；系统按配送日、自提点、线路、商品汇总配送数据。
6. 采购入库：系统或人工生成采购单，审核后生成入库单；收货员接单、扫码、上传质检报告、确认收货、上架，库存增加。
7. 出库配送：配送数据进入波次下放，生成出库单和拣货任务；拣货、缺货标记、复检、打印箱标、扫码装车、确认出仓，库存扣减。
8. 司机配送：司机确认发车，按线路查看门店列表或地图，逐个自提点签到、交接商品、处理异常，并取回退货。
9. 自提点交付：自提点查看配送商品汇总、用户订单、收退货数据，确认到货、处理缺货和售后退货。
10. 售后退款：用户发起仅退款或退货退款，后台审核；退货退款必须完成退货交接和回仓验收后才能退款。
11. 分账提现：订单完成或售后结束后生成佣金明细和分账流水，自提点/供应商申请提现，财务审核打款。
12. 积分运营：后台配置下单返积分、签到积分和过期规则，记录每个用户积分账户和每笔积分流水，过期任务按批次扣减。
13. 报表分析：后台按订单、配送、佣金、积分、售后、统计类目、库存、司机费用等维度查看和导出报表。

## 3. 运营后台功能

### 3.1 商品中心

| 功能 | 详细说明 |
| --- | --- |
| 商品档案管理 | 新建、编辑、启用、停用商品；维护商品名称、品牌、主图、详情图、描述、供应商、采购员、展示类目、统计类目、商品状态。 |
| 商品规格管理 | 一个商品支持多个销售规格；规格需维护规格名称、销售单位、库存单位、条码、包装单位、长宽高、重量，用于销售展示、库存换算和车辆装载校验。 |
| 库存规格绑定 | 商品销售规格可绑定库存规格，支持一对一或一对多换算；例如 1 箱 = 24 盒，出入库按库存规格管理，前台按销售规格展示。 |
| 规格系数转换 | 维护销售规格与库存规格的换算系数，后续采购、入库、出库、盘点、分拣都基于换算关系计算数量。 |
| 三级城市价格 | 每个商品维护一级、二级、三级城市价格；发布时可基于城市价格再做临时调整。 |
| 商品联想词 | 商品保存后根据名称、品牌、规格、类目自动生成搜索联想词，运营可手动新增、修改、停用。 |
| ES 同步 | 商品新增、修改、发布、下架后触发 `product.changed` 事件，异步同步 Elasticsearch，并刷新 Redis 商品缓存。 |
| 榜单标签 | 支持维护“鲜果热卖榜”等榜单标签、排序、排名关系；前台商品详情和榜单页展示。 |
| 商品售罄想要 | 售罄商品允许用户点击“想要”或“降价提醒我”，系统记录用户订阅，商品重新上架或价格变化后通过公众号/消息通知。 |
| 商品编辑弹窗 | 列表中点击“编辑商品”打开编辑区，保存后刷新列表、缓存状态、ES 状态和操作日志。 |

### 3.2 类目中心

| 功能 | 详细说明 |
| --- | --- |
| 展示类目树 | 展示类目用于小程序分类页和商品筛选，支持一级、二级树结构，支持新增一级、添加二级、编辑名称、排序、启停。 |
| 统计类目 | 统计类目用于报表归类和经营分析，当前交互按一级类目展示，支持新增、编辑、启停、查看报表。 |
| 商品绑定类目 | 商品必须绑定展示类目和统计类目；展示类目影响前台可见，统计类目影响销售报表和类目汇总。 |
| 类目停用校验 | 停用类目前需校验是否存在启用子级、在售商品或活动商品，存在关联时禁止直接停用。 |

### 3.3 基础资料

| 功能 | 详细说明 |
| --- | --- |
| 供应商管理 | 维护供应商资料、负责人、手机号、结算信息、供货范围；创建后自动生成供应商小程序账号。 |
| 自提点管理 | 维护自提点名称、地址、负责人、联系电话、所属城市、营业状态、休假状态；创建后生成自提点账号。 |
| 采购员管理 | 维护采购员资料、负责城市、负责供应商、负责商品范围；创建后生成采购员账号。 |
| 城市管理 | 按一级、二级、三级城市维护区域结构，城市用于商品价格、售卖范围、数据隔离和报表汇总。 |
| 城市截团时间 | 每个城市配置截团时间；前台下单和发布配送日计算需使用该规则。 |
| 仓库管理 | 维护采购仓、配送仓、中心仓，绑定城市、负责人、仓库账号和数据权限。 |
| 账号生成 | 供应商、自提点、采购员、仓库等主体创建后写入账号表、角色表和数据权限表，可登录对应小程序/H5。 |
| 主体禁用 | 禁用主体时同步禁用账号，后续商品发布、订单查询、移动端登录按主体状态进行拦截。 |

### 3.4 商品发布

| 功能 | 详细说明 |
| --- | --- |
| 发布模板 | 选择商品创建发布模板，设置发布名称、销售开始/结束时间、配送日期、采购仓、销售库存和活动说明。 |
| 配送日指定 | 支持运营手动指定配送日期；配送日期必须大于销售结束时间。 |
| 自动配送日 | 未指定配送日时，系统按城市截团时间、配送规则、仓库作业周期自动计算。 |
| 售卖范围 | 支持城市白名单、自提点白名单，指定哪些城市或自提点可购买该商品。 |
| 城市价格调整 | 发布时可在商品城市价基础上调整本次发布价格，支持按城市分层设置。 |
| 采购仓指定 | 团购发布必须指定采购仓，发布后用于生成采购需求、采购单和入库计划。 |
| 草稿与确认发布 | 保存草稿不影响前台；确认发布后刷新 Redis 商品列表，触发 ES 同步，小程序首页/分类/活动可见。 |
| 下架与复制 | 已结束或已下架发布可复制为新发布，快速复用商品、白名单、价格和配送规则。 |

### 3.5 权限中心

| 功能 | 详细说明 |
| --- | --- |
| 多端角色 | 支持运营后台、小程序、仓配 H5、WMS 后台等端口角色，角色包括平台、城市、供应商、采购员、自提点、仓库、司机等。 |
| 菜单权限 | 按角色控制后台菜单和移动端功能入口，例如商品中心、订单中心、采购单、入库单、司机配送等。 |
| 按钮权限 | 控制新增、编辑、审核、导出、确认退款、确认出仓、扫码装车、提现审核等操作按钮。 |
| 数据权限 | 按城市、供应商、采购员、自提点、仓库、库区、线路、司机隔离数据。 |
| 登录测试 | 支持测试不同角色登录后的可见菜单、按钮和数据范围。 |
| 权限生效 | 用户重新登录或刷新 token 后权限生效，接口层按数据范围拼接查询条件。 |

### 3.6 订单中心

| 功能 | 详细说明 |
| --- | --- |
| 订单列表 | 按订单号、商品名称、用户手机号、订单状态、自提点、下单时间查询订单。 |
| 订单详情 | 聚合展示用户、手机号、商品、规格、数量、价格、优惠券、支付流水、配送状态、售后状态、佣金状态。 |
| 状态流转 | 支持未支付、已付款/待发货、未支付取消、已支付取消、已发货/待收货、已收货/已完成。 |
| 取消订单 | 未支付订单可用户手动取消或系统超时取消；已支付待发货可发起退款取消；已发货后禁止取消，只能走售后。 |
| 支付流水 | 支付成功后生成支付流水，订单状态更新为已付款，并生成佣金/分账预估。 |
| 导出订单 | 支持按筛选条件导出订单明细，导出任务受当前用户数据权限控制。 |

### 3.7 售后中心

| 功能 | 详细说明 |
| --- | --- |
| 售后申请列表 | 按售后单号、订单号、商品、用户手机号、售后类型、售后状态查询售后。 |
| 售后类型 | 支持仅退款、退货退款、部分退款。 |
| 审核通过 | 校验可退金额，更新为待退款或待退货；仅退款审核通过后可直接发起退款。 |
| 审核驳回 | 记录审核原因，写入售后审核日志，并通知用户。 |
| 确认退货 | 退货退款场景需自提点/司机/仓库完成退货交接，售后状态变为已退货待退款。 |
| 确认退款 | 创建退款流水，调用支付退款接口，回调后更新退款成功或退款失败。 |
| 售后状态 | 待审核、已审核待退货、已审核待退款、已退货待退款、退款成功已审核、退款失败已审核、退款成功已退货、退款失败已退货。 |
| 佣金回冲 | 售后退款会冻结、扣减或生成负向佣金，影响自提点和供应商结算。 |

### 3.8 分账中心

| 功能 | 详细说明 |
| --- | --- |
| 分账方账户 | 管理自提点、供应商、平台、采购员等分账方账户，展示余额、冻结金额、可提现金额。 |
| 佣金明细 | 查看订单、商品、用户、自提点、供应商维度的佣金来源和计算规则。 |
| 提成流水 | 记录佣金产生、冻结、入账、扣减、提现冻结、提现释放等流水。 |
| 分账流水 | 订单完成或售后结束后触发分账，生成分账流水并入账。 |
| 提现申请 | 分账方在小程序提交提现申请，后台展示申请单、金额、账户、状态。 |
| 财务审核 | 财务可审核通过或驳回提现；通过后进入打款流程，驳回后释放冻结金额并记录原因。 |

### 3.9 佣金中心

| 功能 | 详细说明 |
| --- | --- |
| 订单商品分佣 | 按订单商品实付金额拆分自提点佣金、供应商货款、平台服务费、采购员佣金等。 |
| 佣金查询 | 支持按订单号、商品名称、用户手机号、分账方、分账状态、下单时间查询。 |
| 重算佣金 | 售后完成、规则调整或异常修正时可重算佣金，保留历史版本审计记录。 |
| 确认入账 | 后台确认后写入账户流水，更新分账方可提现余额。 |
| 冻结原因 | 订单存在售后、退款、异常履约时佣金可冻结，支持查看冻结原因。 |

### 3.10 营销中心

| 功能 | 详细说明 |
| --- | --- |
| 优惠券管理 | 新增、编辑、发放、停用优惠券，维护有效期、发放数量、使用门槛、库存。 |
| 发放黑白名单 | 支持按城市、自提点配置可发放/不可发放范围。 |
| 使用黑白名单 | 支持按商品配置可使用/不可使用范围。 |
| 活动商品页 | 指定商品池形成活动专题页，支持排序、预览和前台跳转。 |
| 首页弹券 | 用户进入小程序首页时自动弹出可领取优惠券。 |
| 推荐规则 | 基于用户喜好、购买历史、榜单、活动标签推荐商品。 |

### 3.11 积分中心

| 功能 | 详细说明 |
| --- | --- |
| 用户积分账户 | 按用户维度展示可用积分、冻结积分、累计获得、累计消耗、待过期积分和最近过期时间。 |
| 积分流水 | 记录每一次积分获得、扣减、冻结、解冻、售后回退、过期和兑换，保留来源类型、来源单号、操作人和备注。 |
| 下单返积分 | 按订单实付金额计算返积分，支持配置每实付 1 元返 1 积分或自定义比例；支付成功后生成积分流水，订单完成后可转可用。 |
| 签到领取积分 | 用户每日签到领取固定积分，可配置连续签到额外奖励；同一用户同一自然日只能生成一次签到积分。 |
| 积分过期批次 | 每一笔正向积分必须写入独立过期时间 `expire_time` 和剩余可用数量 `remain_points`，后续兑换按过期时间优先扣减。 |
| 过期处理 | 定时任务扫描到期且未消耗完的积分批次，生成 `expire` 类型扣减流水，并更新用户积分账户。 |
| 售后回退 | 订单退款或售后成功后按原返积分批次扣回；若积分已兑换或不足，账户进入异常冻结并提示运营处理。 |
| 积分兑换预留 | 后续积分兑换商品时，生成兑换订单和 `redeem` 扣减流水，关联商品、用户和积分批次。 |

### 3.12 报表中心

| 功能 | 详细说明 |
| --- | --- |
| 订单报表 | 按日期、城市、自提点、供应商、商品统计订单数、销售额、退款额。 |
| 分佣报表 | 按分账方、订单、商品、状态汇总佣金、货款、服务费。 |
| 配送商品汇总 | 按配送日、线路、自提点、商品、SKU、箱码统计配送数量和缺货数量。 |
| 统计类目销售 | 按统计类目汇总销售额、订单数、退款额、商品件数。 |
| 售后订单报表 | 汇总售后数量、退款金额、退款状态、售后原因。 |
| 库存报表 | 查询实时库存、锁定库存、可用库存、库存流水、收发存数据。 |
| 司机费用报表 | 按司机、车辆、线路、配送日统计送货数量、退货数量、距离和应付费用。 |
| 导出任务 | 大数据导出写入异步导出任务，完成后下载 Excel。 |

## 4. 用户小程序 / H5 功能

### 4.1 首页与商品浏览

| 功能 | 详细说明 |
| --- | --- |
| 首页商品列表 | 展示当前城市和当前自提点可售商品，一排一个商品卡片，展示图片、名称、价格、销量、提货日期。 |
| 轮播图 | 首页轮播图可跳转活动专题页或商品列表。 |
| 分类入口 | 首页分类入口跳转分类页，按展示类目筛选商品。 |

## 7. WMS 后台功能

### 7.1 单据流转

| 单据 | 生成来源 | 审核要求 | 作业流程 | 库存影响 |
| --- | --- | --- | --- | --- |
| 采购单 | 供应商提交、采购提交、订单自动生成 | 供应商提交需采购审核；采购提交需提交审核；订单自动生成无需审核 | 采购单生成 → 供应商送货 → 仓库交接 → 关联入库单 | 形成采购入库需求 |
| 入库单 | 采购单入库、仓库其他入库、司机退货入库 | 采购入库无需审核；其他入库需仓库审核；退货入库无需审核 | 指定库区 → 接单 → 扫码 → 质检 → 上架 → 完成 | 库存增加 |
| 配送送货单 | 仓库发起、订单自动生成 | 仓库发起需仓库审核；订单自动生成无需审核 | 配送单生成 → 波次下放 → 分拣 → 缺货/报损 → 出库 → 配送 → 自提点交接 | 配送完成 |
| 出库单 | 仓库其他出库、配送单波次下放 | 其他出库需仓库审核；波次下放无需审核 | 出库单生成 → 扫码装车 → 出库/部分出库 → 扣减库存 → 完成 | 库存扣减 |
| 配送退货单 | 退货订单自动生成 | 无需审核 | 退货单生成 → 司机取回 → 回仓交接 → 生成退货入库单 | 合格入库，不合格报损 |

| 一键下放 | 生成下放批次、出库单、拣货任务并锁定库存。 |
| 作业监控 | 查看线路、拣货、复检、装车、配送进度。 |

## 8. 系统任务与中间件

| 任务/事件 | 触发点 | 处理逻辑 | 中间件/组件 |
| --- | --- | --- | --- |
| 商品同步 | 商品保存、发布、下架 | 生成联想词，刷新 Redis 商品列表，同步 Elasticsearch | Redis、Elasticsearch、MQ |
| 未支付取消 | 订单创建后 15 分钟未支付 | 自动取消订单，释放锁定库存，通知用户 | XXL-Job、MQ |
| 支付提醒 | 未支付订单最后 5 分钟 | 公众号提醒用户完成支付 | XXL-Job、公众号通知 |
| 配送数据生成 | 到达配送数据生成节点 | 按配送日、自提点、线路、商品汇总配送数据 | XXL-Job |
| 优惠券事件 | 领取、使用、退回优惠券 | 更新优惠券库存、用户券状态和订单优惠 | RocketMQ |
| 积分发放 | 订单支付/完成、用户签到 | 计算下单返积分或签到积分，写入积分流水并更新用户积分账户 | RocketMQ、MySQL |
| 积分过期 | 到达积分流水过期时间 | 扫描到期且未消耗完的积分批次，生成过期扣减流水 | XXL-Job |
| 订单完成分账 | 订单完成或售后结束 | 生成佣金明细、分账流水，更新分账方账户 | RocketMQ |
| 分账重试 | 分账失败或入账失败 | 重试分账任务并记录失败原因 | RocketMQ、XXL-Job |
| 导出任务 | 后台导出报表/订单/日志 | 异步生成 Excel 文件并提供下载 | XXL-Job |
| 日志记录 | 所有关键按钮操作 | 写业务日志、操作日志、库存流水或审核日志 | MySQL |

## 9. 核心状态清单

### 9.1 订单状态

| 状态 | 说明 |
| --- | --- |
| 未支付 | 用户已提交订单但未完成支付。 |
| 已付款/待发货 | 支付成功，等待采购、仓配或发货。 |
| 未支付取消 | 未支付订单由用户手动取消或 15 分钟超时自动取消。 |
| 已支付取消 | 已付款但未发货订单取消，需走退款处理。 |
| 已发货/待收货 | 仓库确认出仓后进入待收货。 |
| 已收货/已完成 | 用户或自提点确认收货，订单完成后可触发分账。 |

### 9.2 售后状态

| 状态 | 说明 |
| --- | --- |
| 待审核 | 用户提交售后，等待后台或相关角色审核。 |
| 已审核/待退货 | 退货退款审核通过，等待用户或自提点退货。 |
| 已审核/待退款 | 仅退款审核通过，等待退款执行。 |
| 已退货/待退款 | 仓库或自提点已确认退货，等待退款。 |
| 退款成功/已审核 | 仅退款场景退款成功。 |
| 退款失败/已审核 | 仅退款场景退款失败。 |
| 退款成功/已退货 | 退货退款场景退款成功。 |
| 退款失败/已退货 | 退货退款场景退款失败。 |

### 9.3 WMS 作业状态

| 作业 | 典型状态 |
| --- | --- |
| 采购单 | 草稿、待审核、已审核待到仓、已驳回、已完成、已作废 |
| 入库单 | 待接单、待收货、收货中、质检中、待确认收货、待上架、已上架、已关闭 |
| 拣货任务 | 待领取、待拣货、拣货中、待标记缺货、已拣货、已完成 |
| 复检装车 | 待复检、已复检、箱标已打印、装车中、已出仓、装载异常 |
| 配送单 | 待发车、配送中、门店交接中、门店异常、已完成 |
| 退货交接 | 待司机取回、司机已取待返仓、仓库待验收、已入库、已报损 |
