# 商城项目功能清单

编制日期：2026-06-09

## 1. 功能模块总览

| 序号 | 模块 | 所属端 | 说明 |
| --- | --- | --- | --- |
| 1 | 商品中心 | 运营后台 | 商品基础资料、规格、库存规格、价格、联想词、ES 同步 |
| 2 | 类目中心 | 运营后台 | 展示类目、统计类目、多级类目 |
| 3 | 基础资料 | 运营后台 | 供应商、自提点、采购员、城市、仓库、账号生成 |
| 4 | 商品发布 | 运营后台 | 发布模板、配送日期、城市/自提点白名单、采购仓 |

| 6 | 订单中心 | 运营后台/用户小程序 | 下单、支付、取消、发货、收货、订单状态流转 |
| 7 | 售后中心 | 运营后台/用户小程序/自提点/供应商 | 仅退款、退货退款、部分退款、售后状态流转 |
| 8 | 分账中心 | 运营后台/自提点/供应商 | 佣金账户、提现申请、提成流水、分账流水 |

| 10 | 积分中心 | 运营后台/用户小程序 | 用户积分账户、积分流水、下单返积分、签到积分、积分过期 |
| 11 | 报表中心 | 运营后台 | 订单报表、分佣报表、配送汇总、统计类目销售报表 |

| 14 | 供应商小程序 | 小程序 | 销售数据、售后单、商品信息、佣金、提现、发票、送货单 |
| 15 | WMS 入库 | 仓配小程序/H5/后台 | 采购订单、入库单、收货、质检、上架、库存更新 |
| 16 | WMS 出库 | 仓配小程序/H5/后台 | 销售单、波次下放、拣货、复检、装车、配送 |
| 17 | 库存管理 | 仓配小程序/H5/后台 | 实时库存、库存流水、FIFO、库位、移库、报损、盘点 |
| 18 | 系统任务 | 服务端 | 定时任务、消息队列、缓存刷新、ES 同步、公众号通知 |

## 2. 运营后台功能

### 2.1 商品中心

| 功能 | 说明 |
| --- | --- |
| 商品基础信息 | 维护商品名称、图片、描述、供应商、采购员、类目等基础信息 |
| 商品规格 | 一个商品支持多个规格 |
| 库存规格绑定 | 商品规格可绑定库存规格 |
| 规格系数转换 | 支持商品规格与库存规格之间的换算关系 |
| 城市价格 | 每个商品维护三级城市价格信息 |
| 商品联想词 | 创建商品时自动生成联想词，支持自定义新增和修改 |
| ES 同步 | 商品新增、修改后同步到 Elasticsearch |
| 商品售罄想要 | 记录用户“想要”，商品重新上架后可通知用户 |

### 2.2 类目中心

| 功能 | 说明 |
| --- | --- |
| 展示类目管理 | 创建、编辑、删除展示类目，支持多级 |
| 统计类目管理 | 创建、编辑、删除统计类目，支持多级 |
| 商品绑定展示类目 | 商品用于前台展示和筛选 |
| 商品绑定统计类目 | 商品用于报表统计和经营分析 |

### 2.3 基础资料

| 功能 | 说明 |
| --- | --- |
| 供应商管理 | 创建供应商资料，并生成供应商登录账号 |
| 自提点管理 | 创建自提点，绑定城市，并生成自提点登录账号 |
| 采购员管理 | 创建采购员资料，并生成采购员登录账号 |
| 城市管理 | 维护一二三级城市 |
| 城市截团时间 | 每个城市配置截团时间 |
| 仓库管理 | 创建采购仓/配送仓，并生成仓库登录账号 |

### 2.4 商品发布

| 功能 | 说明 |
| --- | --- |
| 发布模板 | 通过模板发布商品 |
| 配送日期指定 | 支持手动指定配送日期 |
| 配送日期校验 | 配送日期必须大于销售结束时间 |
| 自动计算配送日 | 未指定配送日时系统自动计算 |
| 城市白名单 | 指定某些城市可售 |
| 自提点白名单 | 指定某些自提点可售 |
| 城市价格调整 | 发布时可手动调整商品城市价格 |
| 指定采购仓 | 发布团购必须指定采购仓，衔接采购流程 |
| 商品限购 | 后台发布商品时可设置每人限购数，用户加购和结算均需校验限购 |

### 2.5 权限中心

| 功能 | 说明 |
| --- | --- |
| 用户登录 | 支持平台、城市、供应商、采购员、自提点、仓库等角色登录 |
| 菜单权限 | 按角色控制后台菜单和操作权限 |
| 数据权限 | 按城市、供应商、采购员、仓库、自提点进行数据隔离 |
| 角色管理 | 支持预设角色和自定义角色 |
| 权限配置 | 支持功能权限、数据权限、端口权限配置 |

### 2.6 订单中心

| 功能 | 说明 |
| --- | --- |
| 订单列表 | 查看用户订单，按状态、城市、自提点、商品筛选；支付超时自动取消后展示为“已取消”，保留“再次购买”入口 |
| 订单详情 | 查看商品、价格、数量、取消数量、退货数量、优惠、支付、配送、售后信息，商品操作在价格行右侧展示；订单需记录订单类型、订单备注和下单时提货人快照；提货日延期时展示新的预计提货日，部分退款只在商品行展示“已退款/部分退款” |
| 订单状态流转 | 未支付、已付款、取消、已发货、已收货、已完成 |
| 支付流水 | 支付成功后生成支付流水 |
| 未支付取消 | 15 分钟未支付自动取消，支持用户手动取消 |
| 发货后限制 | 订单发货后不允许取消，只能走售后 |

### 2.7 售后中心

| 功能 | 说明 |
| --- | --- |
| 售后申请列表 | 查看用户提交的售后单 |
| 售后审核 | 审核仅退款或退货退款 |
| 部分退款 | 支持订单部分退款 |
| 仅退款 | 不需要退货时审核后直接退款 |
| 退货退款 | 退货完成后才能退款 |
| 售后状态 | 待审核、待退货、待退款、退款成功、退款失败等 |

### 2.8 分账中心

| 功能 | 说明 |
| --- | --- |
| 分账方账户 | 管理自提点、供应商等分账方佣金账户 |
| 佣金明细 | 查看订单佣金明细 |
| 提成流水 | 记录佣金产生过程 |
| 分账流水 | 订单完成后生成分账流水 |
| 提现申请 | 分账方提交提现申请 |
| 财务审核 | 后台审核提现申请并处理打款 |

### 2.9 营销中心

| 功能 | 说明 |
| --- | --- |
| 优惠券管理 | 创建、发放、停用优惠券；同一张券每个用户只能领取一次，不允许重复领取；优惠券过期未使用后不退回库存；用户券到期前 1 天通过公众号和站内消息提醒一次；整单退款返还优惠券时原券已过期则不返还 |
| 发放黑白名单 | 指定城市或自提点可发放/不可发放 |
| 使用黑白名单 | 指定商品可使用/不可使用优惠券 |
| 活动商品页 | 指定某些商品放在同一活动页面 |
| 推荐规则 | 根据用户喜好推荐商品 |
| 首页弹券 | 用户进入小程序首页时自动弹出优惠券领取；同一用户每天只自动弹出一次；首页如同时存在优惠券弹窗、协议提示、自提点异常提示和活动弹窗，优先级为协议/登录相关 > 自提点异常 > 优惠券 > 活动，未登录用户不弹优惠券信息 |

### 2.10 积分中心

| 功能 | 说明 |
| --- | --- |
| 用户积分账户 | 按用户记录可用积分、冻结积分、累计获得、累计消耗、最近过期积分和最近过期时间 |
| 积分流水 | 每一次获得、扣减、冻结、回退、过期都写流水，保留关联订单、签到或兑换单 |
| 下单返积分 | 按订单实付金额计算返积分，可配置每实付 1 元返 1 积分或自定义比例 |
| 签到领取积分 | 用户端提供签到入口；用户每日签到领取固定积分，可配置连续签到奖励规则；连续签到中断后重新计算连续天数，不允许用户补签 |
| 积分过期时间 | 每一笔正向积分必须记录独立过期时间，后续兑换时按最快过期批次优先扣减 |
| 过期任务 | 定时扫描到期且未消耗完的积分批次，生成过期扣减流水并更新用户账户 |
| 售后回退 | 订单退款或售后完成后按原返积分批次扣回，避免退款后仍保留返积分 |
| 积分兑换商品 | 当前阶段用户端展示积分兑换入口、可兑换商品数据和所需积分；积分当前阶段只用于兑换商品，不做下单金额抵扣；积分兑换订单与普通订单区分，订单号前缀不同并记录订单类型；兑换商品不使用优惠券、不走微信支付通道、不产生各方佣金；提交后立即扣减积分并锁库存，取消后返还积分并释放库存；兑换商品不允许售后，仅允许未出库前取消，出库后不允许用户取消 |

### 2.11 报表中心

| 功能 | 说明 |
| --- | --- |
| 订单报表 | 按时间、城市、自提点、商品统计订单 |
| 分佣报表 | 各分账方订单分佣统计 |
| 配送商品汇总报表 | 按配送日、城市、自提点、商品汇总配送数量 |
| 统计类目销售报表 | 按统计类目汇总销售数据 |
| 售后订单报表 | 统计售后数量、金额、退款状态 |
| 库存报表 | 查询实时库存、库存流水、收发存数据 |

## 3. 用户小程序功能

| 功能 | 说明 |
| --- | --- |
| 既有项保持 | 当前已确认不需要更改的用户端 H5 内容保持不动，本轮只调整自提点、预计提货日、榜单展示、活动页展示、支付倒计时、售后权限和提货人信息等决策项 |
| 首次进入 | 用户第一次进入小程序时先选择城市和自提点；选择后记录上次城市和自提点，下次进入默认使用；用户可手动切换自提点，也可先切换城市后再切换自提点；浏览阶段可不授权登录，进入下单页面或我的、订单、积分、售后、优惠券等个人资产页时，如未授权登录，需先完成授权登录；首次授权登录前必须勾选同意用户协议和隐私协议 |
| 个人资产页登录 | 用户进入“我的、订单、积分、售后、优惠券”等个人资产页时，都必须先登录；未登录时跳转授权登录，登录完成后返回原页面 |
| 用户信息展示 | 用户端“我的”页面展示微信头像昵称；订单和评价不展示头像昵称，降低隐私风险；用户端手机号、提货人、订单等敏感信息不脱敏展示 |
| 首页商品列表 | 展示当前城市/自提点可售商品，首屏突出当前自提点、预计提货日和切换自提点；商品卡片展示图片、名称、规格、当前自提点最终价、剩余库存数、预计提货日和加购入口；剩余数为 0 时显示“已售罄”标记并沉底展示；首页不展示榜单标签，榜单信息只放在榜单页和商品详情页 |
| 轮播图跳转 | 通过轮播图进入活动商品列表 |
| 优惠券弹窗 | 用户进入首页自动弹出可领取优惠券，每天只弹一次；同一张券每个用户只能领取一次，不允许重复领取，优惠券过期未使用后不退回库存；首页弹窗优先级为协议/登录相关 > 自提点异常 > 优惠券 > 活动，未登录用户不弹优惠券信息 |
| 商品详情 | 查看商品规格、当前自提点最终价、剩余库存数、预计提货日、自提点名称、地址、营业/提货时间、可领券入口、自提保障和售后规则摘要等信息；售后期按商品类目配置且需要默认数据，商品类型区分生鲜和普通商品，类目上配置对应售后规则信息；售罄商品详情页允许进入，展示“已售罄 + 到货提醒”；到货提醒仅在商品于同一自提点可售且有库存时触发通知；用户点击别人分享的商品时，如果当前自提点不可售，仍展示商品并提示“当前自提点不可购买”；详情页采用更简洁的小程序风格，弱化销量、购买人数、近期已售等统计展示 |
| 商品分享 | 分享单个商品时只绑定分享人信息；被分享用户打开商品后按当前自提点判断可售范围，不可售时展示商品并提示“当前自提点不可购买” |
| 首页分享 | 分享首页商品列表时只绑定分享人信息 |
| 轮播页分享 | 分享轮播活动页面时只绑定分享人信息 |
| 展示类目列表 | 按展示类目查看商品，首页和分类页类目入口使用短名单行展示，最多显示 5 个字；分类商品卡片展示规格和轻量化商品信息 |
| 商品搜索 | 搜索页支持历史搜索、热门搜索和联想词展示；搜索无结果时展示推荐商品 |
| 购物车 | 同一个购物车只能放同一自提点的商品；购物车有商品时跨城市切换自提点需弹窗确认“切换后将清空购物车”；支持添加、修改、删除购物车商品；购物车里的商品如果价格变化，直接更新为最新价；如果商品已售罄或已下架，沉底置灰显示；用户修改购物车数量时实时校验库存、限购和销售期；优先展示已选商品和结算信息，推荐商品置于商品列表之后 |
| 多提货日结算 | 允许不同预计提货日的商品一起结算，不需要拆分多个订单；同一订单里多个提货日商品一起结算时，结算页按“预计提货日”分组展示，订单列表不展示提货日，订单详情按商品行展示对应提货日；订单仍保持一个主订单。 |
| 失效商品 | 商品过销售期或不满足当前自提点售卖范围时进入已失效状态，失效商品沉底置灰、无法选中，只能删除；切换自提点后不可售商品保留沉底并提示“当前自提点不可购买”；已失效购物车商品超过 7 天后自动清理，同时允许用户手动删除 |
| 活动页商品列表 | 活动页只展示当前自提点可购买商品；活动页商品卡片统一只展示当前自提点最终价、“剩余 + 预计提货日”，不展示已售数量、榜单标签、购买人数等统计信息；剩余数为 0 时显示“已售罄”标记并沉底展示 |
| 切换自提点 | 用户可切换当前自提点；购物车有商品时如果新自提点与当前自提点城市不同，弹窗确认“切换后将清空购物车”；下单时校验商品城市与当前自提点城市是否一致；切换成功后记住新的城市和自提点 |
| 切换城市 | 用户可切换当前城市；跨城市时可先切换城市，再选择该城市下的自提点 |
| 提货人信息 | 只维护提货人姓名和手机号，结算页可修改默认提货人；一个用户最多维护 15 个提货人；允许删除默认提货人，删除后最新添加的提货人变成默认提货人；用户第一次下单时如果没有提货人，使用授权手机号和用户微信名称自动生成一个默认提货人；不维护用户收货地址 |
| 申请自提点 | 用户可申请成为自提点，需填写姓名、手机号、门店名称、地址、备注等信息；提交后进入运营后台审核 |
| 商品想要 | 售罄商品保留“想要/到货提醒”能力；用户订阅后，只有商品在同一自提点可售且有库存时才触发通知；该能力只做到货提醒，不做预约、不锁库存 |
| 我的评论 | 查看并删除自己已发布的商品评论 |
| 商品通知 | 查看自己设置到货提醒的售罄商品，并支持删除通知商品；商品重新上架通知一次后自动删除提醒；只有商品在同一自提点可售且有库存时才触发通知；该入口只做提醒管理，不做预约和锁库存 |
| 优惠券列表 | 查看已领取优惠券，并跳转可用商品列表；优惠券除首页弹窗外，在“我的优惠券”和商品详情/活动页也展示可领券入口；活动价商品允许继续使用优惠券，但需要满足优惠券使用条件；结算页展示优惠券不可用原因，如“不适用当前自提点/商品/金额未达标”；未使用优惠券到期前 1 天通过公众号和站内消息提醒一次；售后退款时整单退款且原优惠券未过期才退还已使用优惠券，原优惠券已过期则不返还，部分退款不退还优惠券 |
