商城项目功能清单与业务流转说明

本文件按“页面 - 按钮 - 点击交互 - 后端处理 - 数据流转 - 状态变化”的方式描述。文档中的每个功能按钮均给出交互和数据流转，适合作为产品原型、接口设计和开发拆分依据。

日期：2026-06-09 项目：商城 + 进销存配 + WMS 后端：Spring Boot / Spring Cloud / Java 18 前端：Vue 3 / TypeScript / Element Plus

1. 范围说明

按钮清单按当前业务范围定义，后续新增页面或按钮应按同一格式补充。

数据表名为建议命名，开发时可按实际数据库规范调整。关键要求是：每次按钮操作都要能落到明确的数据对象、状态变化、操作日志或消息事件。
覆盖端口运营后台、用户小程序、自提点小程序、供应商小程序、仓配小程序/H5、系统任务
覆盖流程商品建档、发布、下单支付、采购入库、出库配送、售后退款、分账提现
数据原则核心业务操作必须记录操作人、操作时间、前后状态、关联单据和消息事件

2. 主业务流转

主流程从基础资料开始，到订单完成后分账结束。

1. 基础资料供应商、自提点、城市、仓库、采购员、类目、权限账号
2. 商品建档商品、规格、库存规格、城市价格、联想词
3. 商品发布发布模板、配送日、白名单、采购仓
4. 用户下单购物车、优惠券、订单、支付流水
5. 采购入库采购单、入库单、收货、质检、库存流水
6. 波次出库销售单、出库单、拣货任务、复检、装车
7. 配送收货司机配送、自提点收货、用户确认
8. 售后退款仅退款、退货退款、退款流水、库存回退
9. 分账提现订单完成、分账流水、佣金账户、提现审核

4. 状态流转说明

状态变化必须由按钮操作、系统任务或支付/退款回调触发。

对象状态流转触发动作
订单未支付 → 已付款待发货 → 已发货待收货 → 已收货已完成提交订单、支付回调、出仓确认、确认收货
取消订单未支付 → 未支付取消；已付款待发货 → 已支付取消取消按钮、超时任务、后台取消
售后单待审核 → 待退货/待退款 → 已退货待退款 → 退款成功/退款失败提交售后、审核、确认退货、退款回调
采购单草稿 → 待审核 → 已审核 → 已入库/部分入库 → 完成保存、提交审核、审核通过、收货完成
出库单待下放 → 待拣货 → 拣货中 → 待复检 → 待装车 → 已出仓波次下放、接单、完成拣货、复检、装车确认
对象状态流转触发动作
商品发布草稿 → 已发布 → 已下架 → 已结束保存草稿、发布、下架、销售结束任务
优惠券未开始 → 进行中 → 已领完/已结束/已停用创建、领取、结束任务、停用
库存可用、锁定、出库、退回、报损下单锁定、取消释放、出库扣减、退货回库、报损
提现待审核 → 审核通过/驳回 → 打款中 → 已打款/打款失败提交提现、财务审核、打款回调
账号启用 → 禁用 → 删除新增主体、禁用账号、删除或停用主体

5. 运营后台：基础资料按钮交互

供应商、自提点、采购员、城市、仓库创建后生成可登录账号。

页面功能按钮点击交互后端处理数据流转状态/校验
供应商列表供应商管理新增供应商打开新增弹窗，填写名称、联系人、手机号、结算信息校验手机号唯一，创建供应商并生成账号写入 base_supplier、sys_user、sys_user_role、sys_data_scope默认启用；重复手机号不可保存
供应商列表供应商管理编辑回显供应商资料，允许修改联系人、结算信息、状态更新供应商资料，记录操作日志更新 base_supplier，写 sys_operation_log已有关联订单时不允许删除关键主体信息
供应商列表供应商管理禁用二次确认，提示禁用后供应商不可登录禁用供应商账号和主体base_supplier.status=disabled，sys_user.status=disabled禁用后不能被新商品选择
自提点列表自提点管理新增自提点打开弹窗，选择城市，填写地址、负责人、手机号创建自提点并生成自提点账号写 base_pickup_station、sys_user、sys_data_scope必须绑定城市；手机号唯一
自提点列表自提点管理休假设置打开休假日历，选择休假日期范围保存休假申请或后台直接设置休假写 station_holiday_record，影响商品可售和配送排期休假日期内用户不可选择该自提点下单
城市列表城市管理新增城市选择上级城市，填写城市名称、等级、截团时间创建城市节点写 base_city城市等级为一/二/三级；截团时间必填
城市列表城市管理编辑截团时间弹出时间选择器，保存后提示影响后续商品售卖更新城市截团时间更新 base_city.cutoff_time，刷新 Redis 城市配置缓存已发布商品不自动改销售结束时间，只影响新发布规则
仓库列表仓库管理新增仓库填写仓库名称、地址、负责人、绑定城市创建仓库和仓库登录账号写 base_warehouse、sys_user、wms_area 默认库区默认生成收货区、存储区、拣货区、退货区等库区
采购员列表采购员管理新增采购员填写姓名、手机号、负责城市/类目创建采购员和登录账号写 base_buyer、sys_user、sys_data_scope商品可绑定采购员；采购员仅查看负责商品
权限中心角色权限保存权限勾选菜单、按钮、数据范围后保存更新角色权限和数据权限写 sys_role_menu、sys_role_button、sys_data_scope，刷新权限缓存用户重新登录或刷新 token 后生效

6. 运营后台：商品与发布按钮交互

商品、类目、发布模板、搜索同步和售卖范围的核心交互。

页面功能按钮点击交互后端处理数据流转状态/校验
商品列表商品管理新增商品进入商品编辑页，录入基础信息、供应商、采购员、类目、图片保存商品主表，生成商品编码写 prd_product；写 sys_operation_log供应商、采购员、展示类目、统计类目必填
商品编辑规格管理新增规格新增规格行，填写规格名、单位、售价、库存规格、转换系数校验规格不重复，保存规格和换算关系写 prd_sku、prd_stock_sku_relation转换系数必须大于 0
商品编辑城市价格批量设置价格选择城市层级，输入价格，批量覆盖或追加生成/更新城市价格记录写 prd_city_price价格不能小于 0；三级城市未配置时继承上级规则
商品编辑搜索词生成联想词点击后展示系统生成词，可手动增删按商品名、别名、类目、规格生成联想词写 prd_suggest_word，发送 MQ 商品同步事件重复词自动去重
商品编辑商品保存保存校验表单，保存后返回列表或停留编辑页事务保存商品、规格、价格、联想词更新 prd_product 等表；发送 product.changed 事件；ES 异步更新保存失败回滚全部修改
商品列表上下架停用商品二次确认，提示已发布商品不自动下架修改商品基础状态prd_product.status=disabled，发送 product.changed停用后不能新发布
展示类目类目维护新增类目选择父级类目，输入名称、排序、状态保存类目树节点写 prd_category(type=display)父级停用时子级不可启用
统计类目类目维护新增类目选择父级类目，输入名称、排序、状态保存统计类目节点写 prd_category(type=stat)用于报表统计，不直接影响前台展示
发布模板商品发布新建发布选择商品、销售时间、配送日、采购仓、售卖范围创建发布模板草稿写 sale_publish_template、sale_publish_product配送日必须大于销售结束时间
发布模板售卖范围设置白名单打开城市/自提点选择器，勾选可售范围保存范围配置写 sale_publish_scope(scope_type=city/station)有白名单时仅白名单范围可见可买
发布模板发布确认发布二次确认，展示销售时间、配送日、采购仓校验库存规格、价格、配送日、采购仓，发布商品sale_publish_template.status=published；写 sale_publish_log；刷新 Redis 商品列表；发送 ES 同步事件发布成功后用户小程序可见
发布列表下架下架输入下架原因并确认修改发布状态，清理前台缓存sale_publish_template.status=offline；写 sale_publish_log；刷新 Redis下架后不可下单，已支付订单继续履约

7. 运营后台：订单与售后按钮交互

订单按钮必须严格受订单状态控制，避免重复支付、重复退款、发货后取消等问题。

页面功能按钮点击交互后端处理数据流转状态/校验
订单列表查询搜索按订单号、用户、城市、自提点、状态、时间筛选按当前用户数据权限查询读取 ord_order、ord_order_item、pay_trade供应商/自提点角色只能看自身数据
订单列表详情查看详情进入订单详情页，展示商品、优惠、支付、配送、售后聚合订单相关数据读取订单、支付、优惠券、配送、售后、分账数据无状态变化
订单详情取消取消订单二次确认，选择取消原因判断状态；未支付直接取消，已支付未发货发起退款取消更新 ord_order.status；释放库存锁定；写 ord_cancel_record；必要时写 afs_refund_trade已发货订单不可取消，只能售后
订单详情备注添加备注弹窗输入内部备注保存备注并记录操作人写 ord_order_remark、sys_operation_log备注不影响订单状态
订单列表导出导出订单选择导出字段和时间范围创建导出任务，异步生成文件写 sys_export_task，读取 ord_order 汇总导出受数据权限控制
售后列表审核审核通过弹窗选择售后类型、退款金额、是否需要退货校验退款金额和订单状态，更新售后状态afs_after_sale.status=approved_wait_return 或 approved_wait_refund；写 afs_audit_log退款金额不能超过可退金额
售后列表审核审核驳回输入驳回原因，确认后通知用户更新售后状态为已驳回afs_after_sale.status=rejected；写 afs_audit_log；发送公众号/小程序通知已退款售后不可驳回
售后详情退货确认退货录入退货数量、收货仓库、退货备注生成退货记录，必要时回补库存写 afs_return_record；写 wms_inventory_log(return_in)；afs_after_sale.status=returned_wait_refund退货退款必须先确认退货再退款
售后详情退款确认退款二次确认退款金额和退款账户调用支付退款接口，创建退款流水写 afs_refund_trade；退款回调更新 afs_after_sale 和 ord_order_refund_amount防重复退款：同一售后只允许一笔进行中退款
售后详情退款失败处理重新退款展示失败原因，确认后重新发起退款校验上一笔退款失败后重新创建退款请求新增 afs_refund_trade；更新原失败流水关联仅退款失败状态可重新退款

8. 分账、营销与报表按钮交互

分账在订单完成后执行，营销配置影响下单优惠和商品可见范围。

页面功能按钮点击交互后端处理数据流转状态/校验
分账账户账户查看查看流水打开流水明细，按订单、时间、类型筛选查询分账和提现流水读取 fin_commission_account、fin_split_flow、fin_withdraw_apply按分账方数据权限过滤
提现审核提现处理审核通过二次确认提现金额和账户冻结金额转打款中，创建付款任务fin_withdraw_apply.status=approved/paying；扣减冻结余额可提现余额必须充足
提现审核提现处理驳回输入驳回原因释放冻结金额，通知分账方fin_withdraw_apply.status=rejected；回写账户余额已打款申请不可驳回
优惠券列表优惠券新增优惠券填写面额、门槛、领取时间、使用时间、数量创建优惠券批次写 promo_coupon_batch、promo_coupon_rule发放数量、有效期、门槛必填
优惠券编辑黑白名单设置名单选择城市、自提点、商品黑白名单保存优惠券使用和发放范围写 promo_coupon_scope黑名单优先级高于白名单
优惠券列表停用停用二次确认，提示未使用优惠券将不可用修改券批次状态promo_coupon_batch.status=disabled；刷新优惠券缓存已使用订单不受影响
活动页配置活动商品添加商品打开商品选择器，支持排序保存活动页商品关系写 promo_activity_product；刷新活动页缓存仅已发布商品可添加
推荐规则商品推荐保存规则配置用户偏好、类目权重、商品权重保存推荐规则，刷新推荐缓存写 promo_recommend_rule；更新 Redis规则启用后影响首页推荐商品
报表中心报表查询选择日期、城市、自提点、供应商、类目聚合统计数据读取订单、售后、分账、配送、库存数据查询范围受数据权限控制
报表中心报表导出选择导出字段，创建导出任务异步生成 Excel写 sys_export_task，生成文件地址大数据量必须异步导出

9. 用户小程序按钮交互

用户端按钮直接影响购物车、订单、支付、售后、优惠券和公众号通知。

页面功能按钮点击交互后端处理数据流转状态/校验
首页城市/自提点切换城市打开城市列表，选择后刷新首页商品保存用户当前城市更新 user_profile.current_city_id；重新读取 Redis 商品列表城市切换后购物车需校验商品是否仍可售
首页城市/自提点切换自提点打开当前城市自提点列表，选择后刷新商品保存用户当前自提点更新 user_profile.current_station_id；刷新商品和配送日休假自提点不可选择
首页优惠券领取优惠券弹窗展示可领券，点击领取校验领取范围、库存、次数写 promo_coupon_user；扣减 coupon_batch.remain_qty不符合城市/自提点名单不可领取
首页/活动页商品浏览查看商品进入商品详情页读取商品发布详情和当前城市价格读取 sale_publish_product、prd_city_price、库存可售数据已下架商品跳转失败并提示
