🟦 入库主流程
🛒
采购订单（PO）
按供应商/商品建立采购计划，WMS 只展示到仓协同、数量和状态，不展示采购价格或结算金额
Admin · 采购管理
📋
入库单（IN）
采购订单审核后自动生成入库单，分配收货库区，状态：draft → approved
Admin 审核 自动生成
📦
收货 / 扫码上架
收货员通过仓配 H5 扫码确认实收、破损、拒收数量，录入批次、生产日期、保质期、质检结果、差异原因和图片凭证；第一版不强制温度和温区校验
H5 receiver H5 收货
🔢
待上架库存
收货完成不等于可售入库，先进入待上架库存；完成上架确认并绑定库位后才进入在库或可售库存
自动同步 Admin 库存查询
🔍
上架 / 库存流水
上架完成后写入库存流水；上架数量小于收货数量时记录去向：继续待上架、异常库存、报损或退供
H5 上架 Admin 查看报告
🟩 出库主流程
🌐
电商订单导入 / 手工销售单
三方电商平台自动同步团长订单（bd_route_delivery），或由内部手工录入销售单（MS...）
ec_sync 自动同步 Admin · 销售单录入
🌊
波次下放
管理员预览未下放订单，选择配送日/线路；一键下放前校验库存锁定、重复下放和 FIFO 批次，失败回滚拣货任务和出库占用
Admin · 波次下放 自动生成出库单+拣货任务
🧺
拣货
拣货任务由系统分配或主管转派；按仓区、通道、库位编码排序，扫码商品码/库位码/任务码确认数量
H5 picker H5 pool/task
✅
缺货联动
缺货是正常履约状态，确认前提示其他库位、同批次和相邻库位；确认后同步订单、自提点汇总和运营履约状态，不生成异常处理单
H5 picker Admin 履约状态
🚛
装车 / 出仓
第一版不做复检模块；装车扫码绑定线路、车辆、司机和箱码，少件、重复、线路或自提点不匹配直接拦截
H5 loading Admin 装车管理
🗺️
司机 / 配送
确认出库后才正式扣减库存并开放司机发车；司机 H5 默认只看当天任务，签到留定位、时间和交接备注
H5 driver Admin 配送看板
⚙️ 辅助与管理模块
📊
库存管理
实时库存查询、库存流水、FIFO 批次解释、移库操作、批次状态和库存锁定释放
🔄
补货管理
仓内补货、采购补货、售后补发三类入口分开；补货员 H5 按系统分配或主管转派执行
📋
即时盘点
全盘 / 差异盘点 / 移动盘点 3 种模式；差异审核后生成库存调整单，二次确认后才生效
🍎
水果仓专区
原料入库 → 生产加工 → 成品入库 → 损耗台账，专属 H5 操作页面
📦
供应商退货
退货回仓按商品行选择可售入库、异常库存、报损、退供，关联售后单、责任方、图片凭证和库存流水
🖨️
打印模板
可视化打印模板设计器，支持采购订单、箱标签、出库单等多场景模板
👥
权限管理 RBAC
收货员、上架员、拣货员、装车员、司机、补货员、盘点员、仓库主管等角色 + 功能权限 + 库区数据权
📈
数据报表
第一版只做作业统计：收货量、拣货量、装车量、缺货量、超时任务和任务完成时长，不做司机费用

移动端

各岗位独立页面模块
