1. 全链路主流程

商品 -> 团期 -> 订单 -> 仓配 -> 自提 -> 售后/分账
基础资料

供应商、自提点、城市、仓库、线路、司机、账号和数据权限。

商品建档

SPU/SKU、规格、类目、供应商、采购员、城市价格和搜索词。

团期发布

销售时间、截团时间、配送日、采购仓、价格、库存模式。

用户浏览

选择城市/自提点后浏览首页、类目、搜索、活动和商品详情。

购物车/下单

校验团期、可售范围、价格快照、库存售卖模式和提货信息。

支付处理

微信支付成功后生成支付单、资金流水和订单已付款状态。

截团汇总

按团期、SKU、自提点、配送日汇总需求，进入采购/库存准备。

采购协同

采购单、供应商送货单、到仓凭证和采购审核记录。

入库收货

入库单、收货接单、扫码实收、质检、差异说明。

上架库存

上架任务完成后库存落到仓库、库区、库位、批次。

配送数据

按配送日、仓库、线路、自提点、SKU 汇总生成配送数据。

波次下放

基于配送数据生成波次单、拣货任务、装车单。

拣货装车

按库位拣货，缺货标记，装车确认；装车不正式扣库存。

确认出库

出库只能扣减一次库存，写库存流水，开放司机任务。

司机配送

司机发车、线路配送、到店签到、配送异常上报。

自提完成

自提点签收，到货通知，用户提货；订单商品进入完成口径。

佣金生成

订单商品已自提后可生成佣金明细。

订单分账

订单主表完成后生成最终分账，非支付后分账。

提现申请

自提点、团长、供应商提交提现，冻结可提现金额。

财务审核

提现不能自动通过；通过后进入打款，驳回释放冻结金额。

售后入口

已发货后禁止取消，只能走仅退款或退货退款。

退款/退货

仅退款审核后直接退款；退货退款需退货确认后退款。

异常中心

缺货、配送异常、退款失败、分账失败、对账差异统一闭环。

报表审计

订单、履约、库存、售后、资金、权限变更全链路留痕。

未支付分支

提交订单后未支付：用户可取消；15 分钟超时关闭；最后 5 分钟公众号提醒。

缺货分支

拣货缺货影响履约，进入缺货处理，可补货、替换、部分退款或关闭。

退货回仓

退货单由退货订单生成，司机取回，仓库交接后合格入库或报损。

第一版暂缓

优惠券、积分中心、箱码、地图线路、复杂 WMS 统计、温区/水果仓不纳入主流程。

2. 分角色泳道

同一业务对象跨多端协同，按角色和数据权限隔离
运营后台
经营驾驶舱

GMV、订单、退款、缺货、履约率和待办异常。

商品团期

商品档案、类目、团期、发布/下架和消息触达。

订单履约

订单中心、履约轨迹、缺货处理。

售后审核

退款、退货退款、责任方认定。

财务风控

资金流水、分账、提现、对账和异常资金。

权限审计

角色、数据权限、导出权限和操作日志。

用户小程序
选择自提点

城市、自提点、休假和可售范围校验。

浏览商品

首页、分类、搜索、活动、详情、评价。

下单支付

购物车、确认订单、微信支付、支付结果。

订单查看

待付款、备货中、待提货、已完成。

申请售后

仅退款、退货退款、售后详情。

消息触达

支付提醒、到货通知、售后通知。

自提点/团长
工作台

今日提货、到店、异常、资金摘要。

门店作业

配送汇总、商品明细、到货通知。

签收协同

司机到点签到后确认到货和待提货。

异常反馈

少件、破损、错货、用户未取。

佣金提现

佣金明细、提现申请、审核结果。

休假申请

影响用户选点和未支付订单继续支付。

供应商
销售摘要

商品销售、采购待办、货款摘要。

采购单

采购明细、审核状态、到仓计划。

送货单

提交预计到仓、送货数量、联系人和凭证。

缺货反馈

可供数量不足、预计补货时间、缺货说明。

货款结算

结算单、扣款、索赔、可提现金额。

操作记录

采购、到仓、提现、缺货记录只读留痕。

WMS / 仓配
基础数据

仓库、库区、库位、线路、司机。

入库

收货、质检、上架、库存流水。

库存

实时库存、锁定、盘点、移库、报损。

出库

波次、拣货、装车、确认出库。

配送

司机任务、发车、签到、回仓。

差异留痕

收货、上架、拣货、装车、回仓差异必须填原因。

3. WMS 单据链与生成规则

采购 -> 入库 -> 上架 -> 波次 -> 拣货 -> 装车 -> 出库 -> 配送 -> 回仓
采购单

供应商发起、采购发起、订单自动生成。

入库单

采购入库、其他入库、退货入库。

上架任务

指定库区库位，库存增加，写入库流水。

配送单

仓库发起或订单自动生成。

波次单

按配送数据下放，生成拣货任务。

出库单

其他出库或配送波次生成。

司机配送单

确认出库后开放司机执行。

回仓确认单

退货取回、仓库交接、库存处理。

单据生成来源审核规则后续动作
采购单供应商发起；采购发起；订单自动生成供应商发起需采购审核和仓库审核；采购发起需仓库审核；订单自动生成无需审核供应商按采购单送货到指定仓库
入库单采购入库；仓库其他入库；司机退货入库采购入库无需审核；仓库自发起需要审核；退货入库无需审核接单、扫码收货、质检、上架、库存增加
配送单仓库发起；订单自动生成仓库发起需仓库审核；订单自动生成无需审核波次下放、拣货、标记缺货、生成出库单
出库单仓库其他出库；配送单波次下放其他出库需仓库审核；波次生成无需审核装车、出库/部分出库、扣减库存
配送退货单根据退货订单自动生成无需审核司机取回退货，回仓后生成退货入库单
库存规则说明
库存定位库存必须落到仓库、库区、库位、批次；可售库存只来自已上架可售存储区。
库存扣减拣货和装车不正式扣库存，确认出库才扣减库存。
幂等提交关键提交动作必须幂等，重复提交只返回第一次有效结果。
差异处理收货、上架、拣货、装车、回仓差异必须填写原因并留痕。
权限边界普通员工只处理分配任务；主管可转派、关闭、查看失败提交。

4. 订单、售后与资金状态

订单主状态、售后状态、资金状态分线管理
订单状态进入条件可执行动作 / 限制
未支付用户提交订单但未支付用户取消、支付、系统超时关闭；后台不提供手动取消。
已付款/待发货支付成功，未出库可按权限取消并退款；取消必须填写原因、二次确认、记录日志。
已发货/待收货出库完成并进入配送/待自提禁止取消，只能走售后。
已完成订单下所有订单商品已自提可售后、评价、分账；售后不回退履约状态。
已关闭未支付超时或用户取消未支付订单只读查看，不产生退款。
已取消已支付未发货被后台取消进入退款链路，需要退款流水。
售后/资金状态流程规则
仅退款用户申请 -> 后台审核 -> 直接退款支持部分退款，退款失败可重试并进入异常中心。
退货退款用户申请 -> 后台审核 -> 退货取回 -> 仓库/运营确认 -> 退款退货进度和退款明细在用户端只读展示。
分佣订单商品已自提后生成佣金明细退款完成后按商品和退款金额扣回佣金。
分账订单主表完成后生成最终分账非支付后分账；失败进入异常资金处理。
提现提交提现 -> 财务审核 -> 打款不能自动通过；驳回释放冻结金额。
对账差异支付、退款、提现、分账对账必须进入异常中心或异常资金处理，不能只在报表展示。

5. 异常入口与第一版边界

高风险动作必须二次确认或留痕
履约异常

缺货、配送延误、无法送达、地址异常、少件、破损、错货进入履约/异常处理；用户商品问题走售后。

资金异常

退款失败、分账失败、提现异常、押金不足、对账差异进入异常资金处理。

高风险操作

发布、下架、取消已支付订单、退款、提现、分账、导出、权限变更、确认出库必须二次确认或留痕。

数据隔离

平台管理员看全局；城市运营看授权城市；供应商、自提点、仓库角色只能看自身或授权范围。

暂缓能力

优惠券、积分中心、箱码、地图线路、复杂 WMS 统计、车辆装载优化、温区/水果仓不进入当前第一版主流程。

文档同步

后续 UI 原型增删页面、调整字段或修改流程时，需要同步追加更新功能清单变更记录。
