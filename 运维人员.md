# Agents 定义文档

> 资深实施工程师 Agent 体系
> 适用场景：云服务器部署、运维、中间件配置、CI/CD 流水线、系统迁移
> 最后更新：2026-06-25

---

## 概述

本文档定义了面向**云服务部署与系统实施**场景的 AI Agent 规格。
每个 Agent 承担独立的实施职责，可单独运行，也可通过编排组合完成完整的上线交付流程。

---

## Agent 列表

---

### 1. 服务器初始化Agent

**角色**：新服务器接手后的第一道工序执行者

**职责**：
- 系统基础配置（时区、主机名、hosts、swap、ulimit）
- 安全加固（禁止 root SSH 登录、修改默认端口、配置 fail2ban）
- 防火墙规则初始化（iptables / firewalld / 安全组同步）
- 基础软件安装（curl、wget、git、vim、htop、lrzsz、lsof、net-tools）
- 创建运维用户并配置免密 sudo
- 配置 NTP 时间同步（阿里云 / 腾讯云 NTP 源）

**支持系统**：CentOS 7/8、Ubuntu 20.04/22.04、Debian 11/12

**输入**：
- 服务器 IP、SSH 端口、用户名
- 目标环境（dev / test / pre / prod）

**输出**：
- 初始化检查报告（Markdown）
- 运维用户 SSH 公钥

**典型脚本片段**：
```bash
# 时区设置
timedatectl set-timezone Asia/Shanghai

# 禁止 root 远程登录
sed -i 's/^PermitRootLogin.*/PermitRootLogin no/' /etc/ssh/sshd_config

# 系统参数优化
cat >> /etc/sysctl.conf <<EOF
net.core.somaxconn = 65535
net.ipv4.tcp_max_syn_backlog = 65535
vm.swappiness = 10
EOF
sysctl -p
```

### 2. Docker 部署Agent

**角色**：容器化应用的部署与生命周期管理者

**职责**：
- Docker Engine 安装（支持离线包安装，适应无外网环境）
- Docker Compose 编排文件生成与执行
- 私有镜像仓库（Harbor）对接配置
- 容器健康检查、重启策略配置
- 日志驱动配置（json-file 滚动限制 / ELK 接入）
- 镜像版本管理与回滚

**支持场景**：
- 单机 Docker Compose（适合 dev/test）
- 多服务编排（后端服务 + Nginx + Redis + MySQL）
- 生产环境蓝绿部署

**docker-compose.yml 模板**：
```yaml
version: '3.8'

services:
  app:
    image: registry.example.com/app:${APP_VERSION}
    container_name: app
    restart: always
    ports:
      - "8080:8080"
    environment:
      - SPRING_PROFILES_ACTIVE=prod
      - JAVA_OPTS=-Xms512m -Xmx1024m
    volumes:
      - ./logs:/app/logs
      - ./config:/app/config
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8080/actuator/health"]
      interval: 30s
      timeout: 10s
      retries: 3
    networks:
      - backend

  nginx:
    image: nginx:1.25-alpine
    container_name: nginx
    restart: always
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/conf.d:/etc/nginx/conf.d
      - ./nginx/ssl:/etc/nginx/ssl
      - ./nginx/html:/usr/share/nginx/html
    networks:
      - backend

networks:
  backend:
    driver: bridge
```

**输出**：
- docker-compose.yml
- .env 环境变量文件
- 部署日志

### 3. Kubernetes 部署Agent

**角色**：K8s 集群应用交付与资源管理者

**职责**：
- Namespace、RBAC、ConfigMap、Secret 初始化
- Deployment / StatefulSet / DaemonSet 编写
- Service / Ingress / HPA 配置
- PVC 存储卷申请与挂载
- Helm Chart 打包与发布
- 滚动更新、金丝雀发布策略配置
- 节点亲和性、污点容忍配置（保障 WMS 服务隔离）

**关键资源模板**：
```yaml
# Deployment 模板
apiVersion: apps/v1
kind: Deployment
metadata:
  name: wms-service
  namespace: production
spec:
  replicas: 3
  selector:
    matchLabels:
      app: wms-service
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1
      maxUnavailable: 0
  template:
    spec:
      containers:
        - name: wms-service
          image: registry.example.com/wms:latest
          resources:
            requests:
              cpu: "500m"
              memory: "512Mi"
            limits:
              cpu: "2000m"
              memory: "2Gi"
          readinessProbe:
            httpGet:
              path: /health
              port: 8080
            initialDelaySeconds: 30
            periodSeconds: 10
---
# HPA 自动扩缩容
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: wms-service-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: wms-service
  minReplicas: 3
  maxReplicas: 20
  metrics:
    - type: Resource
      resource:
        name: cpu
        target:
          type: Utilization
          averageUtilization: 70
```

**输出**：
- Helm Chart 包
- kubectl apply 操作日志
- 资源健康状态检查报告

### 4. Nginx 配置Agent

**角色**：流量入口与反向代理配置专家

**职责**：
- HTTP → HTTPS 强制跳转配置
- 多域名虚拟主机配置
- 反向代理 upstream 负载均衡（轮询 / IP hash / 最少连接）
- SSL 证书申请（Let's Encrypt / acme.sh）与自动续期
- Gzip 压缩、静态缓存、防盗链配置
- 限流限速（limit_req_zone / limit_conn_zone）
- 企业微信、微信支付回调 IP 白名单配置

**Nginx 生产配置模板**：
```nginx
upstream app_backend {
    least_conn;
    server 10.0.0.1:8080 weight=3 max_fails=3 fail_timeout=30s;
    server 10.0.0.2:8080 weight=3 max_fails=3 fail_timeout=30s;
    keepalive 64;
}

server {
    listen 80;
    server_name api.example.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api.example.com;

    ssl_certificate     /etc/nginx/ssl/fullchain.pem;
    ssl_certificate_key /etc/nginx/ssl/privkey.pem;
    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_ciphers         HIGH:!aNULL:!MD5;

    # 限流：每个 IP 每秒最多 20 个请求
    limit_req zone=api_limit burst=50 nodelay;

    location / {
        proxy_pass         http://app_backend;
        proxy_http_version 1.1;
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_set_header   X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_connect_timeout 10s;
        proxy_read_timeout    60s;
    }

    location /static/ {
        root /data/web;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
```

**输出**：
- Nginx 配置文件（按域名分文件）
- nginx -t 验证日志
- SSL 证书到期监控配置

### 5. 中间件部署Agent

**角色**：MySQL、Redis、RabbitMQ、Kafka、Nacos 等中间件的搭建者

**职责**：
- MySQL 主从搭建、半同步复制配置
- Redis 单机 / 哨兵 / Cluster 模式部署
- RabbitMQ 集群搭建、镜像队列配置
- Kafka 集群部署、Topic 规划
- Nacos 集群部署（注册中心 + 配置中心）
- 各中间件监控接入（Prometheus Exporter）

**关键配置原则**：

| 中间件 | 生产必配项 |
|--------|-----------|
| MySQL | 开启 binlog、设置 slow query 阈值 1s、innodb_buffer_pool_size = 内存 70% |
| Redis | 禁用危险命令（FLUSHALL/KEYS）、开启 AOF + RDB、maxmemory-policy allkeys-lru |
| RabbitMQ | 设置 vm_memory_high_watermark 0.4、开启镜像队列、配置死信队列 |
| Kafka | 副本数 ≥ 3、min.insync.replicas=2、消息保留 7 天 |
| Nacos | 使用 MySQL 持久化、开启鉴权、集群节点数奇数 ≥ 3 |

**输出**：
- 安装脚本（Shell）
- 配置文件
- 连通性验证报告

### 6. CI/CD 流水线Agent

**角色**：自动化构建与发布流程搭建者

**职责**：
- Jenkins / GitLab CI / GitHub Actions 流水线配置
- 多环境发布策略（dev 自动发布 / test 手动触发 / prod 二次审批）
- Maven / Gradle / npm / pnpm 构建优化（缓存加速）
- 镜像构建推送（Docker Build + Harbor Push）
- 自动化测试集成（单测 + 接口测试）
- 钉钉 / 企业微信发布通知

**GitLab CI 模板**：
```yaml
stages:
  - build
  - test
  - docker
  - deploy

variables:
  MAVEN_OPTS: "-Dmaven.repo.local=.m2/repository"

build:
  stage: build
  cache:
    paths: [.m2/repository]
  script:
    - mvn clean package -DskipTests -q
  artifacts:
    paths: [target/*.jar]

test:
  stage: test
  script:
    - mvn test

docker-build:
  stage: docker
  script:
    - docker build -t $REGISTRY/$APP_NAME:$CI_COMMIT_SHORT_SHA .
    - docker push $REGISTRY/$APP_NAME:$CI_COMMIT_SHORT_SHA

deploy-prod:
  stage: deploy
  environment: production
  when: manual
  only: [main]
  script:
    - ssh deploy@$PROD_HOST "docker pull $REGISTRY/$APP_NAME:$CI_COMMIT_SHORT_SHA && docker-compose up -d"
```

**输出**：
- 流水线配置文件
- 构建产物归档策略
- 发布通知模板

### 7. 系统迁移Agent

**角色**：老系统迁移上云或跨机房数据搬迁专家

**职责**：
- 迁移前评估：依赖梳理、数据量评估、停机窗口规划
- 数据库迁移：mysqldump / mydumper / DTS 工具选型
- 文件数据迁移：rsync 增量同步、OSS/COS 迁移工具
- DNS 切换方案：低 TTL 预热 → 切换 → 观察 → 确认
- 回滚方案预案（每个步骤均有回退操作）
- 迁移后数据一致性核验

**迁移流程**：

```
阶段一：准备期（迁移前 1 周）
  ├─ 梳理服务依赖图谱
  ├─ 目标环境预搭建与验证
  ├─ 全量数据预迁移（不停机）
  └─ DNS TTL 调低至 60s

阶段二：切割期（停机窗口）
  ├─ 停止写入（关闭入口流量）
  ├─ 增量数据追平（binlog 追赶）
  ├─ 数据一致性校验（行数 + 关键业务字段）
  ├─ 切换 DNS / 修改配置
  └─ 启动新环境 + 烟雾测试

阶段三：观察期（切换后 72h）
  ├─ 监控报警密切关注
  ├─ 旧环境保留不销毁
  └─ 确认无问题后正式下线旧环境
```

**输出**：
- 迁移方案文档
- 数据一致性校验脚本
- 回滚操作手册

### 8. 监控告警Agent

**角色**：系统可观测性搭建者

**职责**：
- Prometheus + Grafana 监控体系搭建
- Node Exporter / JVM Exporter / MySQL Exporter 接入
- 告警规则配置（CPU、内存、磁盘、接口错误率、慢查询）
- 钉钉 / 企业微信 / 短信告警通知配置
- 日志收集：ELK（Elasticsearch + Logstash + Kibana）或 Loki + Grafana
- 链路追踪：SkyWalking / Jaeger 接入

**核心告警规则**：

| 指标 | 告警阈值 | 级别 |
|------|----------|------|
| CPU 使用率 | > 85% 持续 5min | P2 警告 |
| 内存使用率 | > 90% | P1 严重 |
| 磁盘使用率 | > 80% | P2 警告 |
| 接口错误率 | > 1% | P1 严重 |
| MySQL 慢查询 | > 10条/min | P2 警告 |
| Redis 内存 | > maxmemory 80% | P1 严重 |

**输出**：
- Grafana Dashboard JSON
- Prometheus rules.yml
- 告警通知配置

## Agent 协作流程

### 新项目上线标准流程

```
服务器初始化Agent
    ↓ 环境就绪
中间件部署Agent
    ↓ MySQL / Redis / MQ 就绪
Docker 部署Agent 或 K8s 部署Agent
    ↓ 应用服务启动
Nginx 配置Agent
    ↓ 域名 / HTTPS / 反向代理配置完成
CI/CD 流水线Agent
    ↓ 自动化发布流程打通
监控告警Agent
    ↓ 可观测性就绪
    → 交付上线 ✅
```

### 系统迁移流程

```
系统迁移Agent（准备期）
    ↓ 目标环境评估完成
服务器初始化Agent + 中间件部署Agent（并行）
    ↓ 目标环境就绪
系统迁移Agent（切割期）
    ↓ 数据迁移 + DNS 切换
监控告警Agent
    ↓ 全量监控接入
    → 迁移完成 ✅
```

## 通用规范

### 环境分层

| 环境 | 用途 | 资源规格 | 数据策略 |
|------|------|----------|----------|
| dev | 开发联调 | 低配 / 共享 | 随意清理 |
| test | 功能测试 | 中配 / 独立 | 定期重置 |
| pre | 预发布验证 | 接近生产 | 脱敏数据 |
| prod | 生产 | 高可用 / 多副本 | 严格保护 |

### 安全规范

- 所有密码类配置必须走环境变量，禁止硬编码到代码或配置文件
- 生产环境数据库端口不对公网开放，只允许内网 IP 段访问
- SSH 登录必须走密钥认证，禁止密码登录
- 所有外部服务调用走 HTTPS，证书有效期监控告警
- 操作记录留存：重要操作必须记录操作人、时间、执行内容到运维日志

### 变更管理

```
变更申请 → 影响范围评估 → 回滚方案确认 → 变更执行 → 验证 → 关闭
```

- 非紧急变更必须在维护窗口（工作日晚间 22:00-24:00）执行
- 每次变更前备份关键配置与数据
- 核心服务变更必须有第二人在线 review
