# 功能清单-用户小程序

## 模块名称

用户小程序首版后端接口模块，对应 `mall-user`。

## 功能列表

| 功能名称 | 用途描述 | 请求路径 | 入参说明 | 出参说明 |
|----------|----------|----------|----------|----------|
| 首页摘要 | 展示当前用户首页计数 | `GET /api/user/home/summary` | `userId`、`cityId`、`stationId` | 商品数、购物车数、待支付、待自提、未读消息 |
| 前台类目 | 展示用户端商品类目 | `GET /api/user/products/categories` | 分页、`keyword` | 类目分页 |
| 商品列表 | 展示已发布团期商品 | `GET /api/user/products` | 分页、城市、自提点、类目、关键词、配送日 | 商品卡片分页 |
| 商品详情 | 展示商品详情和可售 SKU | `GET /api/user/products/{id}` | 商品ID、`userId`、城市、自提点 | 商品信息、SKU、收藏状态、评价数 |
| 商品评价 | 展示商品评价 | `GET /api/user/products/comments` | 分页、商品、自提点、城市 | 评价分页 |
| 购物车列表 | 展示用户购物车 | `GET /api/user/cart/items` | `userId`、城市、自提点、选中状态、有效状态 | 购物车分页 |
| 加入购物车 | 新增或累加购物车商品 | `POST /api/user/cart/items` | 用户、城市、自提点、团期、商品、SKU、数量 | 购物车项 |
| 更新购物车 | 修改数量或选中状态 | `PUT /api/user/cart/items/{id}` | 购物车ID、用户ID、数量、选中状态 | 购物车项 |
| 移除购物车 | 将购物车项置为失效 | `DELETE /api/user/cart/items/{id}` | 购物车ID、用户ID | 购物车项 |
| 提交订单 | 从选中购物车项生成待支付订单和订单商品快照 | `POST /api/user/orders` | 用户、城市、自提点、提货人、幂等键、购物车项 | 订单主信息 |
| 订单列表 | 展示用户订单 | `GET /api/user/orders` | `userId`、状态、关键词、分页 | 订单分页 |
| 订单详情 | 展示订单主信息 | `GET /api/user/orders/{id}` | 订单ID、`userId` | 订单详情 |
| 取消未支付订单 | 用户取消自己的未支付订单 | `POST /api/user/orders/{id}/cancel` | 订单ID、用户ID、取消原因 | 订单主信息 |
| 订单商品 | 展示用户订单商品 | `GET /api/user/orders/items` | `userId`、状态、关键词、分页 | 订单商品分页 |
| 履约轨迹 | 展示订单履约节点 | `GET /api/user/orders/{id}/tracks` | 订单ID、`userId` | 履约轨迹列表 |
| 创建支付单 | 为未支付订单创建微信支付单 | `POST /api/user/orders/{id}/payments` | 订单ID、用户ID、支付渠道、幂等键 | 支付单信息 |
| 提交售后申请 | 用户对已履约订单商品提交售后申请 | `POST /api/user/after-sales` | 用户ID、订单商品、售后类型、数量、金额、原因 | 售后单 |
| 售后列表 | 展示用户售后单 | `GET /api/user/after-sales` | `userId`、售后类型、状态、关键词、分页 | 售后分页 |
| 售后详情 | 展示售后主信息 | `GET /api/user/after-sales/{id}` | 售后ID、`userId` | 售后详情 |
| 售后商品 | 展示售后商品明细 | `GET /api/user/after-sales/{id}/items` | 售后ID、`userId` | 售后商品列表 |
| 退货记录 | 展示售后退货记录 | `GET /api/user/after-sales/{id}/returns` | 售后ID、`userId` | 退货记录列表 |
| 自提点列表 | 展示可选自提点 | `GET /api/user/stations` | 城市、状态、关键词、分页 | 自提点分页 |
| 自提点详情 | 展示自提点资料 | `GET /api/user/stations/{id}` | 自提点ID | 自提点详情 |
| 消息列表 | 展示用户消息 | `GET /api/user/messages` | `userId`、渠道、已读、发送状态、业务类型、分页 | 消息分页 |
| 消息详情 | 展示消息详情 | `GET /api/user/messages/{id}` | 消息ID、`userId` | 消息详情 |
| 消息已读 | 标记用户消息已读 | `POST /api/user/messages/{id}/read` | 消息ID、`userId` | 消息详情 |

## 业务规则

| 规则 | 说明 |
|------|------|
| 用户归属 | 订单、售后、消息、购物车详情必须校验 `userId` 归属，不允许查看或操作其他用户数据 |
| 数据隔离 | C端用户不走后台 `sys_account` 数据范围拦截，当前模块按 `usr_user.id` 做归属校验 |
| 商品可售 | 商品列表仅展示已发布团期、已审核通过、启用商品、启用 SKU |
| 购物车 | 同用户、同自提点、同团期、同 SKU 重复加入时累加数量 |
| 购物车删除 | 删除购物车项采用置为失效，保留测试状态数据 |
| 提交订单 | 订单提交使用幂等键；只从当前用户、当前城市和自提点的有效选中购物车项生成订单 |
| 未支付取消 | 只有未支付订单允许用户取消；取消后同步关闭待支付支付单 |
| 支付边界 | 当前只创建支付单，不处理微信支付回调、查单补偿和支付成功状态推进 |
| 售后申请 | 售后申请只生成待审核售后单和明细，不直接退款；退款执行留给售后/支付模块 |
| 暂缓能力 | 首版不实现支付成功回调、库存扣减、退款执行、订单完成后分账 |

## 依赖关系

| 依赖 | 说明 |
|------|------|
| `mall-api` | 用户端 DTO/VO |
| `mall-common` | 统一返回、分页、异常、安全过滤、数据权限拦截器 |
| MyBatis Plus | 单表 CRUD、分页和逻辑删除 |
| `Sql/mall.sql` | 复用 `usr_*`、`prd_*`、`sale_*`、`ord_*`、`afs_*`、`msg_*` 表 |
