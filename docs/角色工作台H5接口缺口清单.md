# 角色工作台 H5 接口缺口清单

> 范围：`web/role-workbench-h5`。前端已按原型补齐 UI 和交互，真实接口优先；接口未接通时页面保留可验收状态并提示用户。

| 模块 | 前端动作 | 建议接口 | 当前处理 |
|------|----------|----------|----------|
| 自提点订单 | 查询订单详情 | `GET /station/workbench/orders/{orderNo}` | 前端使用当前主体样例详情兜底 |
| 自提点订单 | 确认提货核销 | `POST /station/workbench/orders/verify-pickup` | 二次确认后写入前端操作记录并提示接口未接通 |
| 自提点作业 | 确认到货 | `POST /station/workbench/delivery-stations/confirm-arrival` | 二次确认后保留交互提示 |
| 自提点作业 | 发送到货通知 | `POST /station/workbench/messages/arrival-notice` | 预览影响用户和商品后提示接口未接通 |
| 自提点作业 | 缺货/异常上报 | `POST /station/workbench/exceptions` | 二次确认后写入前端操作记录 |
| 自提点作业 | 退货司机交接 | `POST /station/workbench/returns/confirm-handover` | 二次确认后保留交互提示 |
| 财务 | 提现申请 | `POST /station/finance/withdraws` | 校验最低提现金额后写入前端操作记录 |
| 财务 | 撤销提现 | `POST /station/finance/withdraws/cancel` | 二次确认后提示接口未接通 |
| 休假 | 提交休假申请 | `POST /station/base/leaves` | 展示影响说明，二次确认后写入操作记录 |
| 休假 | 撤销休假申请 | `POST /station/base/leaves/cancel` | 二次确认后提示接口未接通 |
| 消息 | 单条/分类/全部已读 | `POST /station/messages/read`、`POST /station/messages/read-all` | 前端先更新已读状态 |
| 供应商采购 | 采购单列表/详情 | `GET /supplier/purchases`、`GET /supplier/purchases/{purchaseNo}` | 前端使用当前供应商样例数据兜底 |
| 供应商采购 | 确认供货 | `POST /supplier/purchases/confirm` | 二次确认后写入前端操作记录 |
| 供应商到仓 | 提交到仓凭证 | `POST /supplier/deliveries/arrival-proof` | 二次确认后提示接口未接通 |
| 供应商到仓 | 缺货反馈 | `POST /supplier/deliveries/shortage` | 二次确认后写入前端操作记录 |
| 供应商供货 | 补充供货申请 | `POST /supplier/purchases/supplement` | 二次确认后写入前端操作记录 |
| 供应商结算 | 结算明细 | `GET /supplier/settlements` | 前端按当前送货单生成验收明细 |

## 数据隔离要求

- 团点/自提点视角只能传递 `stationId`、`leaderId`，允许展示提货用户姓名、手机号和备注。
- 供应商视角只能传递 `supplierId`，不展示 C 端用户姓名、手机号和备注。
- 前端所有页面必须从统一角色会话读取 `roleType` 和主体 ID，不再在页面里硬编码业务主体。
