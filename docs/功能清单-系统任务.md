# 功能清单-系统任务

## 模块名称

系统任务（mall-job / XXL-Job）

## 模块边界

第一版只开发 UI 设计和功能清单中会影响订单履约、资金补偿、待办异常、库存预警和搜索补偿的确定性后台任务。积分、优惠券、商品定时上架、商品定时下架不作为第一版定时任务开发。

## 业务规则

1. 每个任务执行前都会写入或复用 `sys_task` 记录，执行中、成功、关闭状态的相同幂等键不重复执行。
2. 任务执行失败时记录 `sys_task.fail_reason`、`next_retry_time`，并写入 `op_exception_record` 供运营后台异常中心处理。
3. 截团任务只处理已发布且已到实际截团时间的团期；仓库库存模式生成仓配任务，虚拟库存模式生成采购单和采购明细。
4. 仓配波次必须匹配启用线路，缺少线路时不生成假线路波次，后续由异常/基础数据配置处理。
5. 退款、分账、消息失败重试只重置失败状态，不伪造第三方调用成功结果。
6. 第一版低库存阈值固定为 10，后续可迁移为按仓库、SKU 或品类配置。

## 暂缓范围

| 暂缓项 | 原因 |
|---|---|
| 积分定时任务 | 积分中心第一版隐藏，不进入订单、售后、财务计算链路 |
| 优惠券定时任务 | 优惠券第一版隐藏，不参与领券、用券、退券和消息触达 |
| 商品定时上架 | 当前商品发布由运营手动发布和团期发布流程控制 |
| 商品定时下架 | 当前下架由团期截团、人工下架或售罄状态控制 |

## 依赖关系

| 依赖模块 | 依赖说明 |
|---|---|
| mall-common | 基础实体、MyBatis Plus、数据权限基础能力 |
| XXL-Job | 调度中心注册执行器并触发 Handler |
| MySQL | 读取和写入订单、团期、采购、仓配、资金、待办、异常、消息、报表、搜索表 |
| 运营后台 | 查看系统任务执行记录、异常中心和待办结果 |
