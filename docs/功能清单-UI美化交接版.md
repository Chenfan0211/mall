# 功能清单-UI美化交接版

> 原型依据：`design/商城项目UI设计.html`
> 生成日期：2026-06-24
> 使用对象：后续负责视觉美化、交互细化、页面重绘的 UI 设计人员。
> 文档定位：本文件只说明当前 UI 原型已有页面、角色、业务信息、关键按钮、状态与风控边界，不代表开始后端编码、建表、接口开发或 SQL 设计。

## 1. 交接结论

当前 UI 原型不是单一商城页面，而是一个社区团购 + WMS 仓配一体的全链路原型。美化时必须按 5 个端分别处理：

| 端 | 当前原型入口 | 美化重点 | 主要角色 |
|----|--------------|----------|----------|
| 运营后台 PC | 运营后台菜单 | 高密度管理台、待办、审批、异常、资金和审计 | 平台管理员、城市运营、商品运营、采购、财务、审计 |
| WMS 仓库后台 | 仓库后台 WMS 菜单 | 单据链、库存状态、仓库作业闭环、主管管理视图 | 仓库主管、采购员、仓管 |
| 用户小程序 H5 | 用户小程序 | 商品浏览、选点、购物车、支付、订单、售后、消息 | C 端用户 |
| 角色工作台 H5 | 角色工作台小程序 | 自提点/供应商工作台、订单作业、货款/佣金、通知和记录 | 自提点、团长、供应商 |
| 仓配移动端 H5 | 仓配移动端 | 按仓库角色隔离任务、扫码/提交、库存、线路配送 | 收货员、采购员、拣货员、装车员、司机、仓库主管 |

UI 美化的核心原则：只优化视觉、布局、层级、组件状态和可读性，不改变业务边界。优惠券、积分、箱码、地图线路、复杂车辆装载、复杂 WMS 统计、水果仓专区仍然是暂缓模块，不能在美化过程中重新露出入口。

## 2. 全局美化约束

| 约束 | 说明 |
|------|------|
| 不新增业务模块 | 当前原型已有 56 个有效导航目标，美化时不要新增未拍板页面。 |
| 不打开隐藏入口 | 用户端优惠券、积分；后台优惠券、积分；WMS 箱码、地图线路、复杂车辆装载等入口保持隐藏或暂缓说明。 |
| 不改业务状态 | 美化不能把待审核、待收货、待上架、已锁定、待出库、待配送、待退款、待提现等状态合并成模糊文案。 |
| 不删高风险确认 | 发布、下架、取消已支付订单、退款、提现、分账、导出、权限变更、确认出库、回仓确认都需要二次确认或操作原因。 |
| 不混端 | 用户小程序不展示后台资金和仓库作业；供应商不展示 C 端用户隐私；司机不展示采购价、销售价、结算金额。 |
| 保留空态/异常态 | 每个列表页至少考虑正常态、空态、加载态、异常态、无权限态。 |
| 保留业务编号 | 订单号、售后单号、采购单号、入库单号、波次单号、装车单号、配送单号、提现单号、对账批次号等必须有清晰展示位置。 |
| 保留时间信息 | 截团时间、配送日、预计到仓、提货时间、审核时间、操作时间不可弱化。 |

## 3. 原型导航总览

### 3.1 运营后台 PC 导航

| 一级组 | 页面 | UI 美化目标 |
|--------|------|-------------|
| 运营驾驶舱 | 运营驾驶舱、待办中心、异常中心、城市排行 | 第一屏展示指标、待办和异常处理入口，适合运营快速判断优先级。 |
| 商品与团期 | 商品中心、类目中心、团期管理、榜单标签 | 强化商品资料与可售团期的关系，避免 UI 上误导为商品上架即售卖。 |
| 交易履约 | 订单中心、售后中心、履约跟踪、缺货处理 | 订单、履约、售后、缺货要能互相跳转，突出当前责任方和下一步动作。 |
| 自提点/团长 | 自提点管理、团长审核、休假管理、签收到店追踪、团长评分 | 突出自提点履约能力、休假影响、用户提货和团长风险。 |
| 供应商/采购 | 供应商管理、采购协同、送货单、供应商售后、供应商结算 | 区分采购协同、WMS 收货和财务结算，避免供应商越权处理入库。 |
| 财务风控 | 分账中心、佣金中心、提现审核、对账中心、异常资金、风控规则 | 所有金额、账户、流水、失败重试和审核动作必须清晰可追溯。 |
| 用户与营销 | 用户中心、评论中心、活动页、消息触达 | 当前只保留用户资料、评论、活动内容和消息，不做优惠券/积分。 |
| 报表与系统 | 报表中心、基础配置、操作日志、系统任务 | 强调只读分析、配置影响范围、日志审计和自动任务补偿。 |

## 4. 运营后台 PC 功能清单

### 4.1 运营驾驶舱

| 页面 | 必须展示的信息 | 关键动作 | 状态/异常 | UI 注意点 |
|------|----------------|----------|-----------|-----------|
| 运营驾驶舱 | 今日 GMV、今日订单数、退款率、缺货率、自提点履约率、待处理异常数 | 查看指标、刷新、进入处理页 | 正常、低于阈值、待处理异常、无异常 | 指标卡不能只做装饰，必须能看出哪些指标可点击下钻。 |
| 待办中心 | 商品审核、团期发布、采购、售后、提现、缺货、异常待办 | 通过、驳回、取消、转交、备注、查看详情 | 待处理、处理中、已完成、已驳回、超时 | 待办按优先级和 SLA 排序，P0 待办要有明显提醒。 |
| 异常中心 | 库存、履约、采购、售后、资金、权限、评价异常 | 处理、关闭、升级、跳转来源单据 | P0/P1、待处理、处理中、已关闭、超时 | 异常中心是处理台，不是普通日志列表。 |
| 城市排行 | 城市 GMV、订单、退款、缺货、自提点履约 | 下钻城市、查看异常 | 授权城市、异常城市、无权限 | 城市运营只看授权城市及下级，不能展示全局敏感数据。 |

风控提醒：驾驶舱只能展示运营汇总，不展示供应商端、自提点端、仓库端专属明细。异常和待办必须能追到具体业务对象。

### 4.2 商品与团期

| 页面 | 必须展示的信息 | 关键动作 | 状态/异常 | UI 注意点 |
|------|----------------|----------|-----------|-----------|
| 商品中心 | 商品名、SPU/SKU、销售规格、库存规格、转换系数、供应商、类目、标签、上下架 | 新建、编辑、提交审核、停用、维护 SKU、查看影响范围 | 草稿、待审核、已启用、停用、资料不完整 | 商品是基础资料，不等于正在售卖；美化时需弱化“立即售卖”误导。 |
| 类目中心 | 前台展示类目、后台统计类目、层级、状态、商品数量 | 新增、编辑、停用、排序 | 启用、停用、有关联商品 | 前台展示类目和统计类目要视觉区分。 |
| 团期管理 | 商品/SKU、城市、大区、销售时间、实际截团时刻、配送日期、采购仓、价格、库存模式 | 新建、编辑、发布前校验、发布、下架、复制旧版 | 草稿、待发布、销售中、已截团、已下架、校验失败 | 团期是真正可售单元，必须突出截团、配送日、采购仓和库存模式。 |
| 榜单标签 | 标签、排序、展示周期、商品状态、城市范围 | 调整排序、发布榜单、停用 | 正常、排名冲突、商品不可售 | 榜单只影响前台展示，不改变商品可售状态。 |

风控提醒：团期发布前必须校验商品状态、SKU、销售规格、库存规格、转换系数、类目、城市、大区截团时间、采购仓、价格、限购、起售数量和库存模式。配送日期必须大于销售结束时间。

### 4.3 交易履约

| 页面 | 必须展示的信息 | 关键动作 | 状态/异常 | UI 注意点 |
|------|----------------|----------|-----------|-----------|
| 订单中心 | 订单号、用户、自提点、订单商品、支付、履约节点、取消记录、售后关联 | 查看详情、取消已支付未发货订单、跳转履约/售后 | 待支付、已付款待发货、已发货待收货、待自提、已完成、已取消 | 订单主状态和订单商品履约状态不能混为一个字段。 |
| 售后中心 | 售后单号、售后类型、退款金额、责任方、凭证、处理记录 | 审核通过、驳回、重试退款、查看订单 | 待审核、退货中、退款中、退款完成、退款失败、已关闭 | 售后状态应独立展示，不回退订单履约状态。 |
| 履约跟踪 | 采购、出库、司机配送、自提点签收、用户自提节点 | 查看节点、处理延误、跳转缺货/售后 | 待采购、待出库、配送中、待自提、异常 | 用时间线表达履约节点，不要只做静态表格。 |
| 缺货处理 | 缺货单号、SKU、团期、自提点、影响订单、责任环节、处理方案 | 补货、替换、部分退款、关闭缺货 | 待处理、补货中、已退款、已关闭 | 必须展示影响范围，不能只显示缺货商品。 |

风控提醒：未支付订单只能用户取消或系统 15 分钟超时关闭。后台只允许取消已支付未发货订单，已发货后禁止取消，只能走售后。

| 签收到店追踪 | 配送日、线路、司机、自提点、待提货订单、售后单 | 查看签到、查看订单、查看配送记录 | 待送达、已签到、待自提、异常 | 当前第一版不要求自提点强制确认签收。 |

风控提醒：供应商不能审核自己创建的采购单，不能看到 C 端用户个人信息，不能处理平台财务动作。

| 司机 | 司机配送工作台 | 当前司机当日线路和退货任务 | 确认出发、到达自提点、完成配送、无法送达、退货带回、历史查询 | 采购价、销售价、结算金额、供应商对账、仓库绩效、仓内全量库存 |
| 仓库主管 | 仓库主管工作台 | 本仓全库区 | 实时库存、待上架、移库、盘点、待确认库存、作业监控、转派、关闭任务 | 其他城市仓数据、财务结算金额 |

### 8.3 仓配关键动作

| 动作 | 触发角色 | 必填信息 | 成功结果 | 失败/异常展示 |
|------|----------|----------|----------|----------------|
| 接单 | 收货员 | 入库单号、收货人、时间 | 任务进入收货中 | 已被接单、无权限、任务关闭 |
| 扫码收货 | 收货员 | 商品条码、实收数量、库区、库位 | 生成收货记录 | 扫码失败可手工补录，必须填原因 |
| 上传质检凭证 | 收货员 | 图片、质检结果、批次说明 | 绑定入库单 | 图片失败、质检异常 |
| 确认收货 | 收货员 | 实收数量、差异原因、质检结果 | 库存进入待上架 | 差异未填原因、重复提交、采购单已关闭 |
| 审核采购单 | 采购员 | 采购单、审核结果、原因 | 采购单进入待送货或驳回 | 无权限、供应商自建单需平台审核 |
| 完成拣货 | 拣货员 | 拣货任务、实际数量、差异原因 | 形成待装车商品明细 | 缺货未说明、扫码错误 |
| 装车 | 装车员 | 装车单、线路、自提点、商品数量 | 装车单进入待确认出库 | 未拣完、司机未绑定 |
| 确认出库 | 装车员 | 装车单、司机、车辆、线路 | 正式扣减库存，司机可见配送任务 | 二次确认失败、库存状态异常 |
| 到达自提点 | 司机 | 配送单、自提点、时间 | 自提点配送状态更新 | 不按线路顺序、无网络 |
| 退货带回 | 司机 | 自提点、SKU、数量、原因 | 生成回仓交接数据 | 数量异常、原因缺失 |
| 回仓确认 | 仓库主管 | 回仓单、商品、数量、库区库位、处理结果 | 生成库存流水 | 退货数据不一致、无来源单据 |
| 移库/盘点 | 仓库主管 | 来源库位、目标库位、实盘数量、差异原因 | 更新库位或生成盘点记录 | 差异未说明、跨仓越权 |

风控提醒：仓配移动端所有提交动作都要考虑弱网、重复点击、扫码失败、手工补录和提交失败回看。重复提交只返回第一次有效结果。

## 9. 跨端核心流程

### 9.1 用户下单到履约

```text
用户选点 -> 商品浏览 -> 购物车 -> 结算 -> 微信支付 -> 订单生成 -> 截团汇总 -> 采购/库存 -> WMS 出库 -> 司机配送 -> 自提点到货通知 -> 用户自提 -> 订单完成
```

UI 美化关注点：

| 节点 | 必须保留的信息 |
|------|----------------|
| 用户选点 | 城市、自提点、营业/休假状态、预计提货时间 |
| 商品浏览 | 当前自提点是否可买、预计提货日、价格、规格 |
| 结算支付 | 商品金额、提货人、自提点、支付倒计时、微信支付状态 |
| 订单履约 | 订单状态、商品状态、配送节点、售后入口 |
| 自提通知 | 到货通知、提货时间、自提点位置、联系电话 |

### 9.2 采购到仓到库存

```text
团期/低库存触发采购 -> 采购审核 -> 供应商送货 -> 收货入库 -> 质检/差异 -> 待上架 -> 主管上架 -> 可售库存
```

UI 美化关注点：

| 节点 | 必须保留的信息 |
|------|----------------|
| 采购单 | 来源、供应商、SKU、数量、仓库、预计到仓 |
| 收货 | 商品条码、计划数量、实收数量、差异原因、质检凭证 |
| 上架 | 库区、库位、实际上架数量、上架回执 |
| 库存 | SKU、仓库、库区、库位、批次、状态、流水 |

### 9.3 出库配送到回仓

```text
上架可售 -> 波次锁定 -> 拣货 -> 装车 -> 确认出库 -> 司机配送 -> 自提点 -> 退货带回 -> 主管回仓确认 -> 库存流水
```

UI 美化关注点：

| 节点 | 必须保留的信息 |
|------|----------------|
| 波次 | 团期、配送日、线路、自提点、可售/锁定库存 |
| 拣货 | 拣货员、库位、计划数量、实际数量、差异 |
| 装车 | 装车单、司机、车辆、线路、自提点、商品明细 |
| 配送 | 线路顺序、到达、完成、无法送达、退货带回 |
| 回仓 | 来源自提点、SKU、数量、原因、处理结果 |

### 9.4 售后与资金

```text
用户申请售后 -> 后台审核 -> 退款/退货退款 -> 退款流水 -> 佣金扣回 -> 对账/异常资金 -> 记录可查
```

UI 美化关注点：

| 节点 | 必须保留的信息 |
|------|----------------|
| 售后申请 | 售后类型、商品、数量、原因、凭证、退款金额 |
| 审核 | 责任方、审核原因、通过/驳回、处理记录 |
| 退款 | 原支付流水、退款流水、失败原因、重试记录 |
| 佣金 | 订单商品佣金、售后扣回、自提点/供应商账户 |
| 对账 | 差异批次、处理结果、调整流水 |

## 10. 角色权限与数据隔离

| 角色 | 可见数据 | 禁止越权 |
|------|----------|----------|
| 平台管理员 | 全局运营、商品、订单、财务、权限、日志 | 无业务越权，但高风险操作仍需确认和留痕。 |
| 城市运营 | 授权城市及下级数据 | 不能看未授权城市资金和用户明细。 |
| 商品运营 | 商品、类目、团期、活动、消息 | 不能处理提现、分账、WMS 实操。 |
| 采购员 | 采购单、供应商协同、到仓结果 | 不能处理库存、装车、司机配送内部作业。 |
| 财务 | 资金流水、提现、分账、对账、异常资金 | 不能修改订单履约和库存实操结果。 |
| 自提点/团长 | 当前自提点订单、用户提货信息、佣金、休假、消息 | 不能看其他自提点订单，不能审核资金。 |
| 供应商 | 自身采购、到仓、商品、货款、操作记录 | 不能看 C 端用户信息，不能处理平台财务动作。 |
| 收货员 | 本仓收货任务和收货区数据 | 不能看采购金额、出库配送、跨仓库存。 |
| 拣货员 | 本人任务和授权库区 | 不能看收货、采购、司机和全仓库存。 |
| 装车员 | 已拣完且分配到月台的装车任务 | 不能看采购、收货、司机费用和箱码能力。 |
| 司机 | 当前司机线路、配送状态、退货带回 | 不能看采购价、销售价、供应商结算、仓内全量库存。 |
| 仓库主管 | 本仓全库区任务、库存、日志 | 不能看其他城市仓和财务结算金额。 |

## 11. UI 美化验收清单

| 验收项 | 验收标准 |
|--------|----------|
| 页面覆盖 | 运营后台、WMS、用户 H5、角色工作台 H5、仓配移动端 H5 的现有页面都能找到对应设计。 |
| 状态完整 | 每个列表至少有正常、空态、异常、无权限、加载中设计。 |
| 操作明确 | 每个高风险按钮有二次确认、原因填写或影响范围提示。 |
| 编号清晰 | 订单、采购、入库、上架、波次、拣货、装车、配送、售后、提现、对账编号都有固定位置。 |
| 角色隔离 | 不同角色页面的信息边界清楚，不展示越权数据。 |
| H5 可操作 | 移动端按钮尺寸、扫码入口、提交状态、失败提示适合单手操作。 |
| 风险可见 | 金额、库存、退款、分账、提现、权限、导出、确认出库等风险动作有明显视觉层级。 |
| 暂缓模块隐藏 | 优惠券、积分、箱码、地图线路、复杂车辆装载、复杂 WMS 统计、水果仓专区不作为当前第一版主入口。 |
| 文案不误导 | 商品中心不写成“直接售卖”，供应商到仓不写成“确认入库”，拣货完成不写成“已扣库存”。 |
| 链路闭环 | 用户下单、采购到仓、出库配送、售后退款、分账提现五条主链路都能在 UI 中串起来。 |

## 12. 暂缓与边界

| 暂缓能力 | 当前处理 |
|----------|----------|
| 用户端优惠券 | 当前隐藏，不参与结算，不展示领券、用券、退券。 |
| 用户端积分 | 当前隐藏，不展示积分余额、积分兑换、积分流水。 |
| 后台优惠券/积分 | 只保留历史数据和日志，不展示常规入口。 |
| 可视化审批流 | 第一版固定审批，不做会签、加签、条件分支编排器。 |
| WMS 箱码/箱标 | 第一版无箱码，装车/配送/回仓按商品/SKU、数量、线路、自提点核对。 |
| 地图线路 | 第一版只做线路、自提点顺序、司机快照，不做地图和路线优化。 |
| 复杂车辆装载 | 第一版不做装载模型、装载试算、司机费用、薪酬和罚款。 |
| 复杂 WMS 统计 | 第一版通过主管工作台、库存流水和操作日志排查，不做绩效排名和复杂报表。 |
| 水果仓专区/温区强制 | 第一版不开放水果仓专区，温度只作备注，不强制上传。 |
