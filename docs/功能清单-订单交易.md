# 功能清单-订单交易

## 模块名称

订单交易首批编码模块，对应 `mall-trade`。

## 功能列表

| 功能名称 | 用途描述 | 请求路径 | 入参说明 | 出参说明 |
|----------|----------|----------|----------|----------|
| 查询订单列表 | 运营后台订单中心按订单号/提货人、用户、城市、自提点、团长、交易状态、支付状态、履约状态筛选订单 | `GET /api/trade/orders` | `keyword`、`userId`、`cityId`、`stationId`、`leaderId`、`tradeStatus`、`payStatus`、`fulfillStatus`、`pageNum`、`pageSize` | 订单分页列表 |
| 查看订单详情 | 查看订单主表、交易状态、支付状态、履约状态、金额、提货人快照和取消原因 | `GET /api/trade/orders/{id}` | 订单ID | 订单详情 |
| 查询订单状态日志 | 查看订单交易、支付、履约三条状态线的变更记录 | `GET /api/trade/orders/{id}/status-logs` | 订单ID | 状态日志列表 |
| 查询订单商品明细 | 按订单、团期、商品、SKU、供应商、仓库、城市、自提点、履约状态、售后状态筛选订单商品 | `GET /api/trade/order-items` | `orderId`、`periodId`、`productId`、`skuId`、`supplierId`、`warehouseId`、`cityId`、`stationId`、`fulfillStatus`、`afterSaleStatus`、`pageNum`、`pageSize` | 订单商品分页列表 |
| 查看订单商品详情 | 查看订单商品快照、数量、已提货数量、退款数量、履约状态和预计提货日 | `GET /api/trade/order-items/{id}` | 订单商品ID | 订单商品详情 |
| 查询履约轨迹 | 查看订单商品从支付、采购、入库、配送、待自提到已自提的履约节点 | `GET /api/trade/fulfillment-tracks` | `orderId`、`orderItemId`、`nodeType`、`status` | 履约轨迹列表 |
| 查询缺货记录 | 按团期、订单、订单商品、SKU、仓库、自提点和状态筛选履约缺货记录 | `GET /api/trade/stockouts` | `periodId`、`orderId`、`orderItemId`、`skuId`、`warehouseId`、`stationId`、`status`、`pageNum`、`pageSize` | 缺货记录分页列表 |
| 查看缺货详情 | 查看单条缺货原因、缺货数量、已处理数量和状态 | `GET /api/trade/stockouts/{id}` | 缺货记录ID | 缺货详情 |

## 业务规则

| 规则 | 说明 |
|------|------|
| 只读起步 | 本批只提供查询能力，不实现下单、支付、取消、退款、分账等高风险动作 |
| 状态分线 | 订单主表保留交易状态、支付状态、履约状态三条线，避免把取消、退款、发货、自提、分账混在一个状态里 |
| 明细维度 | 订单商品是履约、售后、退款、分佣、报表统计的关键维度，同一订单下不同商品可有不同履约状态 |
| 售后分离 | 订单商品履约状态不混入售后状态；订单中心只展示 `afterSaleStatus` 作为关联状态 |
| 数据隔离 | 订单主表按城市、自提点、团长范围隔离；订单商品按城市、供应商、自提点、仓库范围隔离；履约轨迹按城市、自提点、仓库范围隔离；缺货记录按自提点、仓库和供应商关联订单商品隔离 |

## 依赖关系

| 依赖 | 说明 |
|------|------|
| `Sql/mall.sql` | 使用 `ord_order`、`ord_order_item`、`ord_order_status_log`、`ord_fulfillment_track`、`ord_stockout_record` 表和初始化数据 |
| `mall-common` | Result、分页、基础实体、全局异常、网关认证头、数据权限拦截器 |
| `mall-api` | 订单、订单商品、履约轨迹、缺货记录 VO/DTO 契约 |
| MyBatis Plus | 单表查询、分页和数据权限插件 |
| MapStruct | Entity 到 DTO 转换 |
