# 功能清单-运营待办与异常中心

## 模块名称

运营待办与异常中心首批编码模块，对应 `mall-operation`。

## 功能列表

| 功能名称 | 用途描述 | 请求路径 | 入参说明 | 出参说明 |
|----------|----------|----------|----------|----------|
| 运营驾驶舱摘要 | 汇总待处理、处理中、超时待办和高优先级异常，并返回前 5 条预警数据 | `GET /api/operation/dashboard/summary` | 无 | 待办统计、异常统计、紧急待办、异常预警 |
| 查询待办列表 | 运营驾驶舱、待办中心按状态、优先级、责任人、城市、供应商、仓库筛选待办 | `GET /api/operation/todos` | `todoType`、`priority`、`status`、`sourceBizType`、`processAccountId`、`cityId`、`supplierId`、`warehouseId`、`pageNum`、`pageSize` | 待办分页列表 |
| 查看待办详情 | 查看单个待办的业务来源、标题、优先级、处理人和截止时间 | `GET /api/operation/todos/{id}` | 待办ID | 待办详情 |
| 查询异常列表 | 异常中心按异常类型、来源模块、优先级、责任对象和状态筛选异常 | `GET /api/operation/exceptions` | `exceptionType`、`priority`、`status`、`sourceModule`、`sourceBizType`、`cityId`、`supplierId`、`stationId`、`warehouseId`、`ownerAccountId`、`pageNum`、`pageSize` | 异常分页列表 |
| 查看异常详情 | 查看单个异常的来源对象、责任模块、责任主体、状态和描述 | `GET /api/operation/exceptions/{id}` | 异常ID | 异常详情 |

## 业务规则

| 规则 | 说明 |
|------|------|
| 只读起步 | 本批只提供查询能力，不绕过订单、库存、资金、售后等原业务模块直接处理状态 |
| 驾驶舱支撑 | 待办和异常列表支持按优先级、状态、责任对象筛选，用于运营驾驶舱待办任务和异常预警 |
| 数据隔离 | 已接入统一数据权限拦截器，按城市、供应商、仓库范围注入过滤；无授权范围的受管业务表默认不可见 |
| 状态口径 | 待办状态使用 `10待处理 20处理中 30已处理 40已驳回 50已取消 60已超时 70已关闭` |
| 异常口径 | 异常状态使用 `10待处理 20处理中 30已处理 40关闭`，第一版异常中心不替代普通售后审核和对账 |

## 依赖关系

| 依赖 | 说明 |
|------|------|
| `Sql/mall.sql` | 使用 `op_todo`、`op_exception_record` 表和已初始化测试数据 |
| `mall-common` | Result、分页、基础实体、全局异常 |
| `mall-api` | 待办/异常 VO、DTO 契约 |
| MyBatis Plus | 单表分页和详情查询 |
| MapStruct | Entity 到 DTO 转换 |
