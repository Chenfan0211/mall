# 功能清单-商城全链路

> 原型依据：`design/商城项目UI设计.html`
> 生成日期：2026-06-24
> 当前阶段：UI 交互原型设计阶段。本文件只梳理功能、页面、拟定接口与业务规则，不代表已经开始后端编码、建表或 SQL 设计。

## 1. 模块名称

社区团购商城全链路系统，覆盖商品、团期发布、用户下单、采购到仓、仓储履约、配送、自提、售后、分账提现、权限审计等核心链路。

本次功能清单按 UI 原型拆为 5 个端：

| 端 | 范围 | 主要角色 |
|----|------|----------|
| 运营后台 PC | 经营驾驶舱、商品团期、订单履约、售后、供应商采购、财务风控、权限报表 | 平台管理员、城市运营、商品运营、采购、财务、审计 |
| WMS 仓库后台 | 采购到仓、收货入库、上架、库存、波次、拣货、装车出库、配送、回仓、日志 | 仓库主管、采购员、仓管 |
| 用户小程序 H5 | 浏览、选点、搜索、购物车、下单、支付、订单、评价、售后、消息 | C 端用户 |
| 角色工作台 H5 | 自提点/团长作业、供应商到仓协同、商品与资金记录 | 自提点、团长、供应商 |
| 仓配移动端 H5 | 收货、拣货、复检/装车、司机配送、主管移动处理 | 收货员、拣货员、复检/装车员、司机、仓库主管 |

## 2. 全局约定

| 项 | 约定 |
|----|------|
| 接口口径 | 下表中的请求路径均为后续代码阶段的拟定路径，当前仅用于功能清单和原型对齐。 |
| 返回结构 | 后续接口统一返回 `Result<T>`，成功 `Result.success(data)`，失败 `Result.fail(ErrorCode)`。 |
| 分页口径 | 列表入参统一包含 `pageNum`、`pageSize`，出参统一包含 `total`、`list`。 |
| 数据隔离 | 平台管理员看全局；城市运营看授权城市及下级；供应商只看自身；自提点只看自身；仓库角色只看授权仓库。 |
| 敏感字段 | 运营后台按权限明文展示和明文导出，风险控制依靠角色权限、数据权限、导出权限和操作日志。 |
| 审计留痕 | 审批、导出、退款、提现、分账、权限变更、仓库关键作业、任务关闭、转派、失败提交必须写操作日志。 |
| 暂缓模块 | 优惠券、积分中心、复杂流程编排、复杂 WMS 统计、箱码、地图线路、水果仓专区属于暂缓或二期能力。 |

## 8. 核心业务规则汇总

### 8.1 订单状态机

| 状态 | 进入条件 | 可执行动作 | 限制 |
|------|----------|------------|------|
| 未支付 | 用户提交订单但未支付 | 用户取消、支付、系统超时关闭 | 后台不提供手动取消按钮 |
| 已付款/待发货 | 支付成功，未出库 | 后台按权限取消并退款、等待履约 | 取消必须填写原因、二次确认、记录日志 |
| 已发货/待收货 | 出库完成并进入配送/待自提 | 查看履约、用户待自提 | 禁止取消，只能走售后 |
| 已完成 | 订单下所有订单商品已自提 | 售后、评价、分账 | 售后不回退履约状态 |
| 已关闭 | 未支付超时或用户取消未支付订单 | 只读查看 | 不产生退款 |
| 已取消 | 已支付未发货被后台取消 | 退款链路、取消记录 | 需要退款流水 |

### 8.2 售后规则

| 类型 | 流程 | 规则 |
|------|------|------|
| 仅退款 | 用户申请 -> 后台审核 -> 直接退款 | 支持部分退款，退款失败可重试并进入异常中心 |
| 退货退款 | 用户申请 -> 后台审核 -> 退货/取回 -> 仓库/运营确认 -> 退款 | 退货进度和退款明细在用户端只读展示 |
| 售后责任 | 运营认定责任方，可关联供应商售后 | 不改变订单商品履约状态 |

### 8.3 商品与团期规则

| 项 | 规则 |
|----|------|
| 商品 | 商品是基础资料，维护 SPU、SKU、规格、类目、供应商和标签。 |
| 团期 | 团期是可售单元，必须配置销售城市、大区、销售时间、配送日期、采购仓、价格、限购、起售数量和库存模式。 |
| 截团 | 实际截团时刻由销售结束日期和大区截团时间生成。 |
| 下单 | 用户必须在销售开始到实际截团之间下单，且当前自提点可买。 |
| 快照 | 下单生成商品名、规格、价格、库存售卖模式、团期、生产日期、保质期、供应商、采购仓/履约仓快照。 |

### 8.4 WMS 规则

| 项 | 规则 |
|----|------|
| 单据链 | 采购单 -> 入库单 -> 上架任务 -> 波次单 -> 拣货任务 -> 装车单 -> 出库单 -> 司机配送单 -> 回仓确认单 -> 库存流水。 |
| 库存 | 库存必须落到仓库、库区、库位、批次；可售库存只来自已上架可售存储区。 |
| 扣减 | 拣货和装车不正式扣库存，确认出库才扣减库存。 |
| 幂等 | 关键提交动作必须幂等，重复提交只返回第一次有效结果。 |
| 差异 | 收货、上架、拣货、装车、回仓差异必须填写原因并留痕。 |
| 权限 | 普通员工只能处理分配任务；主管可转派、关闭、查看失败提交。 |

### 8.5 财务与分账规则

| 项 | 规则 |
|----|------|
| 分佣 | 订单商品已自提后可生成佣金明细。 |
| 分账 | 订单主表完成后才生成最终分账，非支付后分账。 |
| 退款扣回 | 售后退款完成后按退款商品和退款金额扣回佣金。 |
| 提现 | 提现需财务审核，不能自动通过。 |
| 对账 | 支付、退款、提现、分账差异必须进入异常中心或异常资金处理。 |

## 10. 暂缓与边界

| 模块/能力 | 当前处理 |
|-----------|----------|
| 优惠券 | 当前运营后台和用户端入口隐藏；历史数据保留，不进入当前第一版流程。 |
| 积分中心 | 当前隐藏，不落 UI、不落权限、不落系统任务；后续重新开启需重新拍板。 |
| 可视化审批流 | 第一版只做固定审批，不做流程编排、会签、加签、条件分支。 |
| WMS 箱码/箱标 | 第一版不考虑箱码，装车、出库、配送、回仓按商品/SKU、数量、线路、自提点核对。 |
| 地图线路 | 第一版线路配送按线路、自提点顺序和司机快照展示，地图线路为二期。 |
| 车辆装载 | 复杂装载模型和装载优化为二期。 |
| 复杂统计绩效 | WMS 第一版只做基础查询和过程统计，不做复杂报表、绩效排名、异常分析。 |
| 水果仓专区/温区 | 第一版不做水果仓专区，不强制温度，不做温区校验。 |

## 11. 验收关注点

| 验收项 | 说明 |
|--------|------|
| 原型覆盖 | 功能清单必须能对应 `design/商城项目UI设计.html` 的后台页面、WMS 页面和 H5 页面。 |
| 角色隔离 | 不同端和角色看到的数据范围必须与权限表一致。 |
| 链路闭环 | 用户下单 -> 截团 -> 采购/库存 -> WMS -> 配送 -> 自提 -> 售后 -> 分账提现需能串起来。 |
| 高风险动作 | 发布、下架、取消已支付订单、退款、提现、分账、导出、权限变更、仓库确认出库必须二次确认或留痕。 |
| 暂缓入口 | 优惠券、积分、箱码、地图线路、复杂 WMS 统计不能作为当前第一版必做能力出现。 |
| 文档同步 | 后续 UI 原型增删页面、调整字段或修改流程时，需要同步追加更新本功能清单的变更记录。 |
