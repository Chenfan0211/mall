## 发布包生成

在项目根目录执行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File deploy\scripts\package-warehouse-h5-release.ps1
```

脚本会：

- 构建 `web/warehouse-h5`
- 生成 `deploy/release/warehouse-h5-release-YYYYMMDD-HHMMSS.zip`
- 带上 `Sql/warehouse-h5-test-data-20260629.sql`
- 带上线上验收脚本 `verify-warehouse-h5.ps1`
- 校验构建包包含真实登录、仓配 session、司机配送/退货 tab、司机地图模式、司机作业统计、收货记录、拣货记录、采购数量编辑器、数据源标识和自定义底部导航

## 服务器发布

优先使用已验证 release zip 直接部署，避免服务器 Node/npm 环境差异导致重新构建出错。

Windows 发布机只发布 H5：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File deploy\scripts\deploy-warehouse-h5-release.ps1 -ReleaseZip deploy\release\warehouse-h5-release-20260629-235806.zip
```

Windows 发布机发布 H5 并同时导入 SQL：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File deploy\scripts\deploy-warehouse-h5-release.ps1 `
  -ReleaseZip deploy\release\warehouse-h5-release-20260629-235806.zip `
  -ImportSql `
  -RemoteSqlImportCommand 'docker exec -i mall-mysql mysql -uroot -p"$MYSQL_ROOT_PASSWORD" mall'
```

Windows 发布机发布、导库并自动复验：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File deploy\scripts\deploy-warehouse-h5-release.ps1 `
  -ReleaseZip deploy\release\warehouse-h5-release-20260629-235806.zip `
  -ImportSql `
  -RemoteSqlImportCommand 'docker exec -i mall-mysql mysql -uroot -p"$MYSQL_ROOT_PASSWORD" mall' `
  -VerifyAfterDeploy
```

Linux/macOS 发布机只发布 H5：

```bash
DEPLOY_HOST=192.168.28.242 DEPLOY_USER=root bash deploy/scripts/deploy-warehouse-h5-release.sh deploy/release/warehouse-h5-release-20260629-235806.zip
```

Linux/macOS 发布机发布 H5 并同时导入 SQL：

```bash
IMPORT_SQL=1 \
REMOTE_SQL_IMPORT_COMMAND='docker exec -i mall-mysql mysql -uroot -p"$MYSQL_ROOT_PASSWORD" mall' \
DEPLOY_HOST=192.168.28.242 \
DEPLOY_USER=root \
bash deploy/scripts/deploy-warehouse-h5-release.sh deploy/release/warehouse-h5-release-20260629-235806.zip
```

Linux/macOS 发布机发布、导库并自动复验：

```bash
IMPORT_SQL=1 \
VERIFY_AFTER_DEPLOY=1 \
REMOTE_SQL_IMPORT_COMMAND='docker exec -i mall-mysql mysql -uroot -p"$MYSQL_ROOT_PASSWORD" mall' \
DEPLOY_HOST=192.168.28.242 \
DEPLOY_USER=root \
bash deploy/scripts/deploy-warehouse-h5-release.sh deploy/release/warehouse-h5-release-20260629-235806.zip
```

如果发布机没有 Chrome，可保留接口和账号强校验，跳过浏览器复验：Windows 增加 `-SkipBrowserVerify`，Linux/macOS 增加 `SKIP_BROWSER_VERIFY=1`。

如果需要在发布机上重新构建，再执行：

```bash
DEPLOY_HOST=192.168.28.242 DEPLOY_USER=root bash deploy/scripts/deploy-warehouse-h5.sh
```

如果服务器不是 `root` 用户，替换 `DEPLOY_USER`。

## 线上复验

如果需要一次性定位当前阻塞，先运行诊断脚本：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File deploy\scripts\diagnose-warehouse-h5.ps1
```

部署和导库后再运行强校验：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File deploy\scripts\verify-warehouse-h5.ps1
```

推荐在有 Chrome 的发布机上运行完整复验，完整复验会先检查线上新版包、6 个角色账号和 WMS 接口，再打开 Chrome 做 375px 手机尺寸页面、导航、记录、采购编辑、入库回执和装车回执自动验收：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File deploy\scripts\verify-warehouse-h5-full.ps1
```

默认强校验会验证新版 H5 包、WMS 五个只读接口，以及采购员/主管备用真实账号接口兜底。若已经导入完整 SQL，需要额外验证 6 个独立角色账号：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File deploy\scripts\verify-warehouse-h5.ps1 -StrictRoleAccounts
```

验收通过标准：

- `/warehouse/` 最新包包含 `/auth/password-login`
- `/warehouse/` 最新包包含 `driver-map-mode`、`driver-stats-summary`、`warehouse-receive-record-card`、`warehouse-pick-record-card`、`purchase-qty-stepper`、`receipt-gallery`、`receipt-stamp`、`data-source-chip`、`warehouse-bottom`
- 默认校验：收货员、拣货员、装车员、司机真实账号可登录，采购员/主管在专属账号未导入时可通过备用真实 token 拉接口数据
- 严格校验：加 `-StrictRoleAccounts` 后 6 个仓配测试账号均可登录
- 6 个角色登录后工作台显示“接口数据”；采购员和主管在正式账号未导入时会使用备用真实 token 拉接口，但最终仍建议导入 `test_wms_buyer`、`test_wms_manager` 完成角色账号闭环
- 本地自动验收必须在 Chrome Network 里看到每个角色至少 1 个成功的 `/api/wms/*` 响应；主管需要看到入库、上架、库存、拣货、配送 5 类接口
- 本地自动验收必须能从“我的”打开收货员收货记录和拣货员拣货记录，并渲染筛选栏和记录列表
- 本地自动验收必须能进入采购员采购单详情，渲染仓库/日期表单、数量步进器，并验证采购计划数量可调整
- 本地自动验收必须能进入入库回执和装车回执详情，分别渲染回执图片画廊和装车回执章
- WMS 五个只读接口 `total > 0`
- 未登录访问 WMS 接口返回 `401/403`

## 常见坑

- 只替换 `index.html` 不替换 `assets` 会导致页面仍加载旧分包。
- 只部署 H5 不导 SQL，采购员和主管仍会进入演示数据。
- uni-app 会拆分 JS 分包，验收脚本必须递归读取 chunk，不能只检查入口 JS。
- uni-app H5 会把 `<button>` 渲染成 `<uni-button>`；浏览器自动验收点击详情页按钮时必须按实际 DOM 检查。
- 当前仓库只有 `deploy/.env.example`，没有真实 `.env`，不能从代码仓库推断生产 MySQL 密码。
- `192.168.28.242:22` 和 `:80` 网络可达，但 SSH 认证失败；`3306` 直连超时，MySQL 大概率只在服务器或 Docker 内网开放。
